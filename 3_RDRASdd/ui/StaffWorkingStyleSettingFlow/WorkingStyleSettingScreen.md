# UI仕様書: 働き方設定画面

## 1. 概要
**画面日本語名**: 働き方設定画面  
**画面英語名**: WorkingStyleSettingScreen  
**所属BUC**: StaffWorkingStyleSettingFlow  

**目的**: サービススタッフが柔軟な働き方希望を詳細に設定・入力。複数の条件を組み合わせて管理。

---

## 2. 利用アクターと役割
| アクター | 役割・責務 |
|---------|---------|
| サービススタッフ | 働き方希望を設定・入力 |

---

## 3. 関連UC
- **UC-001: スタッフ働き方希望を設定する** (3行目に記載)

---

## 4. 表示/入力項目

| 項目名 | 型 | 必須 | 編集可否 |
|-------|---|-----|--------|
| スタッフID | number | - | - |
| 働き方区分 | enum | ○ | ○ |
| 勤務開始日 | date | ○ | ○ |
| 勤務形態詳細 | string | - | ○ |

---

## 5. API利用（UCエンドポイント）

**設定**: POST /api/v1/staff-working-style/register

**リクエスト**:
```json
{
  "staffId": "number",
  "workingStyleType": "enum",
  "startDate": "string (YYYY-MM-DD)",
  "details": "string (optional)"
}
```

**レスポンス (201)**:
```json
{
  "workingStyleId": "number",
  "staffId": "number",
  "workingStyleType": "string",
  "workingStyleState": "登録",
  "createdAt": "datetime"
}
```

---

## 6. 画面状態

| 状態 | 説明 |
|-----|------|
| 初期表示 | 働き方設定フォーム |
| 入力中 | ユーザーが情報を入力 |
| 設定中 | ボタン無効化 |
| 設定成功 | 成功メッセージ |
| 設定失敗 | エラーメッセージ |

---

## 7. 操作フロー

```
START → [フォーム表示]
  → [働き方区分、開始日、詳細を入力]
  → [設定ボタン押下]
  → POST /register
    ├→ [成功] → 成功メッセージ
    └→ [失敗] → エラーメッセージ
  → [キャンセル] → 前画面へ
  → END
```

---

## 8. 機能要求

| 要求ID | 要求内容 | 優先度 |
|-------|--------|-------|
| FR-001 | 働き方希望を設定できること | 必須 |
| FR-002 | 複数の条件を組み合わせて設定できること | 推奨 |

---

## 9. バリデーション

| 項目 | 検証ルール |
|-----|---------|
| 働き方区分 | 列挙値のみ |
| 勤務開始日 | YYYY-MM-DD、本日以降 |

---

## 10. 監査/権限制御

**権限**: 「サービススタッフ」（ログイン済みユーザー自身のみ設定可能）

**監査ログ**: 働き方設定 → staffId, workingStyleType, startDate, タイムスタンプ
