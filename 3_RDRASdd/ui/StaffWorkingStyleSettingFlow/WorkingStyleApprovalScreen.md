# UI仕様書: 働き方承認画面

**画面日本語名**: 働き方承認画面  
**画面英語名**: WorkingStyleApprovalScreen  
**所属BUC**: StaffWorkingStyleSettingFlow  

**目的**: 審査済みの働き方を確認して最終承認判定を実行。スタッフへの通知内容を決定。

---

## 表示/入力項目

| 項目名 | 型 | 必須 | 編集可否 |
|-------|---|-----|--------|
| スタッフID | number | - | - |
| 働き方区分 | enum | - | - |
| 承認結果 | enum | ○ | ○ |
| 承認日 | date | ○ | ○ |

---

## API利用

**承認**: PATCH /api/v1/staff-working-style/:styleId/approve

**リクエスト**:
```json
{
  "approvalResult": "enum (承認|却下)",
  "approvalDate": "string (YYYY-MM-DD)"
}
```

---

## 操作フロー

```
START → [審査済み働き方一覧表示]
  → [働き方選択] → 詳細表示
  → [承認結果、承認日を入力]
  → [承認ボタン押下] → PATCH /:id/approve
    ├→ [成功] → 成功メッセージ
    └→ [失敗] → エラーメッセージ
  → END
```

---

## 機能要求

| 要求ID | 要求内容 | 優先度 |
|-------|--------|-------|
| FR-001 | 審査済み働き方を一覧表示 | 必須 |
| FR-002 | 承認判定を実行 | 必須 |
