# UI仕様書: 働き方審査画面

**画面日本語名**: 働き方審査画面  
**画面英語名**: WorkingStyleReviewScreen  
**所属BUC**: StaffWorkingStyleSettingFlow  

**目的**: 設定された働き方希望の実現可能性を判定し、審査結果を記録。

---

## 表示/入力項目

| 項目名 | 型 | 必須 | 編集可否 |
|-------|---|-----|--------|
| スタッフID | number | - | - |
| 働き方区分 | enum | - | - |
| 審査結果 | enum | ○ | ○ |
| 審査日 | date | ○ | ○ |

---

## API利用

**審査**: PATCH /api/v1/staff-working-style/:styleId/review

**リクエスト**:
```json
{
  "reviewResult": "enum (適正|要相談|保留)",
  "reviewDate": "string (YYYY-MM-DD)"
}
```

---

## 操作フロー

```
START → [登録済み働き方一覧表示]
  → [働き方選択] → 詳細表示
  → [審査結果、審査日を入力]
  → [審査ボタン押下] → PATCH /:id/review
    ├→ [成功] → 成功メッセージ
    └→ [失敗] → エラーメッセージ
  → END
```

---

## 機能要求

| 要求ID | 要求内容 | 優先度 |
|-------|--------|-------|
| FR-001 | 登録済み働き方を一覧表示できること | 必須 |
| FR-002 | 審査結果を記録できること | 必須 |

---

## 監査/権限制御

**権限**: 「スタッフ管理担当者」

**審査ログ**: 働き方審査 → staffId, reviewResult, タイムスタンプ
