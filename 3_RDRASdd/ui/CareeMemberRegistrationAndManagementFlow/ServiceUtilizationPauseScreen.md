# UI仕様書: サービス利用休止画面

## 1. 概要
**画面日本語名**: サービス利用休止画面  
**画面英語名**: ServiceUtilizationPauseScreen  
**所属BUC**: CareeMemberRegistrationAndManagementFlow  

**目的**: サービス利用中の介護会員のサービス利用を一時中断し、会員状態を「サービス休止」に遷移。休止理由と予定再開日を記録。

---

## 2. 利用アクターと役割
| アクター | 役割・責務 |
|---------|---------|
| 介護会員管理担当者 | 会員のサービス利用休止手続きを実施 |

---

## 3. 関連UC
- **UC-005: サービス利用を休止する** → UC-005

---

## 4. 画面目的
サービス利用中の会員のサービスを一時中断。休止開始日、休止理由、予定再開日を記録し、会員状態を「サービス休止」に遷移。

---

## 5. 表示/入力項目

| 項目名 | 型 | 必須 | 編集可否 | 説明 |
|-------|---|-----|--------|------|
| 会員ID | number | - | - | 表示専用 |
| 会員名 | string | - | - | 表示専用 |
| 利用休止日 | date | ○ | ○ | YYYY-MM-DD |
| 休止理由 | string | ○ | ○ | 1-500文字 |
| 予定再開日 | date | - | ○ | YYYY-MM-DD（オプション） |

---

## 6. コンポーネント（GUI要素）

| コンポーネントID | 種類 | 対応項目 |
|----------------|------|--------|
| txtSearchId | テキスト入力 | 会員ID |
| btnSearch | ボタン | 検索 |
| lblMemberName | ラベル | 会員名 |
| dtPauseDate | 日付ピッカー | 休止開始日 |
| txaPauseReason | テキストエリア | 休止理由 |
| dtResumeDate | 日付ピッカー | 予定再開日 |
| btnPause | ボタン | 休止実行 |
| btnCancel | ボタン | キャンセル |

---

## 7. API利用（UCエンドポイント）

**状態遷移**: POST /api/v1/caremembers/:careeMemberId/service/pause

**リクエスト**:
```json
{
  "pauseStartDate": "string (YYYY-MM-DD)",
  "pauseReason": "string",
  "estimatedResumeDate": "string (YYYY-MM-DD, optional)"
}
```

**レスポンス (200 OK)**:
```json
{
  "careeMemberId": "number",
  "careeMemberState": "サービス休止",
  "pauseStartDate": "string",
  "pauseReason": "string",
  "transitionedAt": "datetime"
}
```

**エラーハンドリング**:
- **404 Not Found**: 会員が見つからない
- **409 Conflict**: 状態が「サービス利用中」でない

---

## 8. 画面状態

| 状態 | 説明 |
|-----|------|
| 初期表示 | 検索フォーム |
| 検索結果表示 | 会員情報表示 |
| 入力中 | 休止日、理由入力 |
| 処理中 | ボタン無効化 |
| 成功 | 成功メッセージ |
| 失敗 | エラーメッセージ |

---

## 9. 操作フロー

```
START → [会員ID入力] → [検索] → GET /:careeMemberId
  → [会員情報表示]
  → [休止日、理由、予定再開日入力]
  → [休止ボタン押下] → POST /service/pause
    ├→ [成功] → 成功メッセージ
    └→ [失敗] → エラーメッセージ
  → [キャンセル] → 前画面へ
  → END
```

---

## 10. 機能要求

| 要求ID | 要求内容 | 優先度 |
|-------|--------|-------|
| FR-001 | 会員を検索できること | 必須 |
| FR-002 | 休止日、理由を入力できること | 必須 |
| FR-003 | 予定再開日（オプション）を入力できること | 推奨 |
| FR-004 | 状態が「サービス利用中」でない場合はエラー表示 | 必須 |

---

## 11. 非機能要求

| 要求ID | 要求内容 |
|-------|--------|
| NFR-001 | 検索結果表示は3秒以内 |
| NFR-002 | 冪等性を持つこと |

## 12. バリデーション

| 項目 | 検証ルール |
|-----|---------|
