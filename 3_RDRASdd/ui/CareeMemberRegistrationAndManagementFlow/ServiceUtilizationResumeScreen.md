# UI仕様書: サービス利用再開画面

## 1. 概要
**画面日本語名**: サービス利用再開画面  
**画面英語名**: ServiceUtilizationResumeScreen  
**所属BUC**: CareeMemberRegistrationAndManagementFlow  

**目的**: サービス休止中の介護会員のサービスを再開し、会員状態を「サービス利用中」に遷移。

---

## 2. 関連UC
- **UC-006: サービス利用を再開する**

---

## 3. 画面目的
サービス休止中の会員を検索・選択し、再開日を指定して、会員状態を「サービス利用中」に遷移。

---

## 4. 表示/入力項目

| 項目名 | 型 | 必須 | 編集可否 |
|-------|---|-----|--------|
| 会員ID | number | - | - |
| 会員名 | string | - | - |
| 利用再開日 | date | ○ | ○ |

---

## 5. API利用（UCエンドポイント）

**状態遷移**: POST /api/v1/caremembers/:careeMemberId/service/resume

**リクエスト**:
```json
{
  "resumeDate": "string (YYYY-MM-DD)"
}
```

**レスポンス (200 OK)**:
```json
{
  "careeMemberId": "number",
  "careeMemberState": "サービス利用中",
  "memberStatus": "サービス利用中",
  "transitionedAt": "datetime"
}
```

**エラーハンドリング**:
- **404**: 会員が見つからない
- **409**: 状態が「サービス休止」でない

---

## 6. 画面状態

| 状態 | 説明 |
|-----|------|
| 初期表示 | 検索フォーム |
| 検索結果表示 | 会員情報表示 |
| 日付入力 | 再開日入力可能 |
| 遷移中 | ボタン無効化 |
| 遷移成功 | 成功メッセージ |
| 遷移失敗 | エラーメッセージ |

---

## 7. 操作フロー

```
START → [会員ID入力] → [検索] → [会員情報表示]
  → [再開日入力] → [再開ボタン押下]
  → POST /service/resume
    ├→ [成功] → 成功メッセージ
    └→ [失敗] → エラーメッセージ
  → [キャンセル] → 前画面へ
  → END
```

---
