# UI仕様書: サービス終了画面

## 1. 概要
**画面日本語名**: サービス終了画面  
**画面英語名**: ServiceTerminationScreen  
**所属BUC**: CareeMemberRegistrationAndManagementFlow  

**目的**: サービス利用中またはサービス休止中の介護会員のサービス利用を終了し、会員状態を「サービス終了」に遷移。

---

## 2. 関連UC
- **UC-007: サービス終了を処理する**

---

## 3. 画面目的
会員を検索・選択し、終了日と終了理由を指定して、会員状態を「サービス終了」に遷移。

---

## 4. 表示/入力項目

| 項目名 | 型 | 必須 | 編集可否 |
|-------|---|-----|--------|
| 会員ID | number | - | - |
| 会員名 | string | - | - |
| 終了日 | date | ○ | ○ |
| 終了理由 | string | ○ | ○ |

---

## 5. API利用（UCエンドポイント）

**状態遷移**: POST /api/v1/caremembers/:careeMemberId/service/terminate

**リクエスト**:
```json
{
  "terminationDate": "string (YYYY-MM-DD)",
  "terminationReason": "string"
}
```

**レスポンス (200 OK)**:
```json
{
  "careeMemberId": "number",
  "careeMemberState": "サービス終了",
  "memberStatus": "サービス終了",
  "terminationDate": "string",
  "transitionedAt": "datetime"
}
```

**エラーハンドリング**:
- **404**: 会員が見つからない
- **409**: 既に終了処理済み、または状態遷移不可

---

## 6. 画面状態

| 状態 | 説明 |
|-----|------|
| 初期表示 | 検索フォーム |
| 検索結果表示 | 会員情報表示 |
| 入力中 | 終了日、理由入力 |
| 遷移中 | ボタン無効化 |
| 遷移成功 | 成功メッセージ |
| 遷移失敗 | エラーメッセージ |

---

## 7. 操作フロー

```
START → [会員ID入力] → [検索] → [会員情報表示]
  → [終了日、理由入力] → [終了ボタン押下]
  → POST /service/terminate
    ├→ [成功] → 成功メッセージ
    └→ [失敗] → エラーメッセージ
  → [キャンセル] → 前画面へ
  → END
```

---

## 8. 機能要求

| 要求ID | 要求内容 | 優先度 |
|-------|--------|-------|
| FR-001 | 会員を検索できること | 必須 |
| FR-002 | 終了日、理由を入力できること | 必須 |
| FR-003 | 状態遷移を実行できること | 必須 |
| FR-004 | 既に終了済みの場合はエラー表示 | 必須 |

---

## 9. バリデーション

| 項目 | 検証ルール |
|-----|---------|
| 終了日 | YYYY-MM-DD、本日以降 |
| 終了理由 | 1-500文字、非空 |

---

## 10. 監査/権限制御

**権限**: 「介護会員管理担当者」

**監査ログ**: 状態遷移操作 → careeMemberId, terminationDate, terminationReason, タイムスタンプ
