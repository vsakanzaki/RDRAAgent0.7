# UI仕様書: サービス利用開始画面

## 1. 概要
**画面日本語名**: サービス利用開始画面  
**画面英語名**: ServiceUtilizationStartScreen  
**所属BUC**: CareeMemberRegistrationAndManagementFlow  
**所属業務**: 介護会員管理業務  

**目的**: 情報確認済みの介護会員がサービス利用を開始する状態遷移を実行。会員状態を「情報確認」から「サービス利用中」へ遷移させる。

---

## 2. 利用アクターと役割
| アクター | 役割・責務 |
|---------|---------|
| 介護会員管理担当者 | 会員のサービス利用開始手続きを実施 |

---

## 3. 関連UC
- **UC-004: サービス利用を開始する** → UC-004

---

## 4. 画面目的
情報確認済みの会員を検索・選択し、サービス利用開始日を指定して、会員の状態を「サービス利用中」に遷移。

---

## 5. レイアウト方針
**セクション構成**:
- **検索セクション**: 会員ID検索
- **確認セクション**: 会員基本情報表示
- **操作セクション**: 利用開始日入力
- **フッター**: 実行ボタン、キャンセルボタン

---

## 6. 表示/入力項目

| 項目名 | 型 | 必須 | 編集可否 | 説明 |
|-------|---|-----|--------|------|
| 会員ID | number | - | - | 検索結果表示 |
| 会員名 | string | - | - | 表示専用 |
| 現在状態 | enum | - | - | 表示専用: 「情報確認」のみ対象 |
| 利用開始日 | date | ○ | ○ | YYYY-MM-DD形式 |

---

## 7. コンポーネント（GUI要素）

| コンポーネントID | 種類 | 対応項目 |
|----------------|------|--------|
| txtSearchId | テキスト入力 | 会員ID |
| btnSearch | ボタン | 検索実行 |
| lblMemberName | ラベル | 会員名 |
| lblCurrentState | ラベル | 現在状態 |
| dtServiceStartDate | 日付ピッカー | 利用開始日 |
| btnStart | ボタン | 開始処理 |
| btnCancel | ボタン | キャンセル |

---

## 8. API利用（UCエンドポイント）

**検索**: GET /api/v1/caremembers/:careeMemberId

**状態遷移**: POST /api/v1/caremembers/:careeMemberId/service/start

**リクエスト**:
```json
{
  "serviceStartDate": "string (YYYY-MM-DD)"
}
```

**レスポンス (200 OK)**:
```json
{
  "careeMemberId": "number",
  "careeMemberState": "サービス利用中",
  "memberStatus": "サービス利用中",
  "transitionedAt": "datetime"
}
```

**エラーハンドリング**:
- **404 Not Found**: 会員が見つからない
- **409 Conflict**: 状態が「情報確認」でない

---

## 9. 画面状態

| 状態 | 説明 |
|-----|------|
| 初期表示 | 検索フォーム |
| 検索結果表示 | 会員情報表示 |
| 日付入力 | 利用開始日入力可能 |
| 遷移中 | ボタン無効化 |
| 遷移成功 | 成功メッセージ |
| 遷移失敗 | エラーメッセージ |

---

## 10. 操作フロー

```
START
  ↓
[会員ID入力]
  ↓
[検索ボタン押下] → GET /:careeMemberId
  ↓
[会員情報表示]
  ↓
[利用開始日入力]
  ↓
[開始ボタン押下] → POST /service/start
  ├→ [成功 (200)] → 成功メッセージ表示
  └→ [失敗] → エラーメッセージ表示
  ↓
[キャンセル] → 前画面へ戻る
  ↓
END
```

## 11. 機能要求

| 要求ID | 要求内容 | 優先度 |
|-------|--------|-------|
| FR-001 | 会員を検索できること | 必須 |
| FR-002 | 利用開始日を指定できること | 必須 |
| FR-003 | 状態遷移を実行できること | 必須 |
| FR-004 | 状態が「情報確認」でない場合はエラー表示すること | 必須 |

---

## 12. 非機能要求

| 要求ID | 要求内容 |
|-------|--------|
| NFR-001 | 検索結果表示は3秒以内 |
| NFR-002 | 状態遷移は冪等性を持つこと |
