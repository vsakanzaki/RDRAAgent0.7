# UI仕様書: 介護会員登録画面

## 1. 概要

**画面日本語名**: 介護会員登録画面  
**画面英語名**: CareMemberRegistrationScreen  
**所属BUC（日本語）**: 介護会員登録・管理フロー  
**所属BUC（英語）**: CareeMemberRegistrationAndManagementFlow  
**所属業務**: 介護会員管理業務  

**目的**: 介護会員管理担当者が受け付けた会員情報を確認して正確性を検証し、システムに正式登録する。登録後、会員状態は「情報確認」となり、スケジュール計画の対象となる。

---

## 2. 利用アクターと役割

| アクター | 役割・責務 | 操作目的 |
|---------|---------|---------|
| 介護会員管理担当者 | 受け付け情報を確認・検証してシステムに登録。入力値の正確性チェックを実施 | 会員情報の確認・検証・登録 |

## 3. 関連UC

- **UC-002: 介護会員情報を登録する**  
  受け付けた会員情報をシステムに正式登録し、会員状態を「情報確認」に遷移。  
  参照: `3_RDRASdd/application/CareeMemberRegistrationAndManagementFlow.md` § 4. UC-002

## 4. 画面目的

受け付けた会員情報を表示し、管理担当者が内容を確認・検証した上でシステムに正式登録。会員ID、会員状況は自動設定される。

---

## 5. レイアウト方針

**セクション構成**:
- **ヘッダー**: 画面タイトル「新規介護会員登録確認」
- **メインセクション**: 会員情報確認フォーム（受け付け情報を表示）
- **フッター**: 「登録」ボタン、「キャンセル」ボタン

**読み順**: 上から下へ。情報確認フローのため、入力フィールドと表示専用フィールドを明確に区別。

## 8. API利用（UCエンドポイント）

**メソッド/パス**: POST /api/v1/caremembers/register

**用途**: 受け付けた会員情報をシステムに正式登録

**リクエストボディ**:
```json
{
  "careeMemberId": "number",
  "name": "string",
  "address": "string",
  "phoneNumber": "string",
  "birthDate": "string (YYYY-MM-DD)",
  "facilityType": "enum"
}
```

**レスポンス (200 OK)**:
```json
{
  "careeMemberId": "number",
  "name": "string",
  "address": "string",
  "phoneNumber": "string",
  "birthDate": "string",
  "facilityType": "string",
  "memberStatus": "契約待ち",
  "careeMemberState": "情報確認",
  "updatedAt": "datetime"
}
```

**エラーハンドリング**:
- **404 Not Found**: 指定した会員IDが見つからない  
  → UI概要: 「この会員は見つかりません。受付情報を確認してください。」
- **409 Conflict**: 既に登録済み  
  → UI概要: 「既に登録済みの会員です。」

## 9. 画面状態

| 状態 | 説明 |
|-----|------|
| 初期表示 | 前画面の受け付け情報を表示。全て入力可能 |
| 検証エラー | バリデーション失敗時、エラーメッセージを表示 |
| 登録中 | API呼び出し中。送信ボタン無効化 |
| 登録成功 | 成功メッセージ表示。詳細管理画面へ遷移 |
| 登録失敗 | エラーメッセージ表示。情報は保持 |

---

## 10. 操作フロー

```
START
  ↓
[前画面からの受け付け情報表示]
  ↓
[ユーザーが情報を確認・修正] ← 必要に応じて入力値を修正
  ↓
[登録ボタン押下]
  ├→ [バリデーション失敗] → エラー表示
  └→ [バリデーション成功]
    ↓
  [API: POST /register] → 送信ボタン無効化
    ↓
    ├→ [成功 (200)] → 成功メッセージ → 管理画面へ遷移
    └→ [失敗 (4xx/5xx)] → エラーメッセージ表示
  ↓
[キャンセル] → 前画面へ戻る
  ↓
END
```

---

## 16. 実装AI向け指示

**状態管理**:
- 前画面の受け付け情報をバッピング（sessionStorage等）で受け取り
- フォーム入力値と自動設定値を分離管理
- 登録処理中は送信ボタンを無効化

**API呼び出し**:
1. 「登録」ボタン押下
2. フロント側バリデーション
3. POST /api/v1/caremembers/register 呼び出し
4. 成功時は詳細管理画面へ遷移
5. 失敗時はエラー表示
