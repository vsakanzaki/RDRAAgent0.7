# UI仕様書: 介護会員情報管理画面

## 1. 概要
**画面日本語名**: 介護会員情報管理画面  
**画面英語名**: CareMemberInformationManagementScreen  
**所属BUC**: CareeMemberRegistrationAndManagementFlow  
**所属業務**: 介護会員管理業務  

**目的**: 登録済みの介護会員情報を検索・表示し、必要な項目を編集・更新できる。変更履歴を管理し、サービス利用状態に応じた更新制限を実施。

---

## 2. 利用アクターと役割
| アクター | 役割・責務 | 操作目的 |
|---------|---------|---------|
| 介護会員管理担当者 | 登録済み会員情報の検索・表示・編集・更新を実施 | 会員情報の管理・保守 |

---

## 3. 関連UC
- **UC-003: 介護会員情報を更新する**  
  参照: `3_RDRASdd/application/CareeMemberRegistrationAndManagementFlow.md` § 4. UC-003

---

## 4. 画面目的
会員一覧から対象会員を検索・選択し、詳細情報を表示・編集。状態に応じた更新制限を実施しながら情報を保守する。

---

## 5. レイアウト方針
**セクション構成**:
- **検索セクション**: 会員ID、施設タイプでフィルタリング
- **一覧セクション**: 該当会員を表示（ページング対応）
- **詳細セクション**: 選択会員の全情報を編集可能で表示

**読み順**: 検索 → 一覧選択 → 詳細編集

---

## 8. API利用（UCエンドポイント）

**初期表示・検索**: GET /api/v1/caremembers?memberStatus={status}&facilityType={type}&page={page}&limit={limit}

**詳細表示**: GET /api/v1/caremembers/:careeMemberId

**更新**: PATCH /api/v1/caremembers/:careeMemberId

**エラーハンドリング**:
- **404 Not Found**: 会員が見つからない → UI概要: 「会員が見つかりません」
- **409 Conflict**: サービス休止中のため更新不可 → UI概要: 「サービス休止中は更新できません」

---

## 9. 画面状態

| 状態 | 説明 |
|-----|------|
| 初期表示 | 検索フォーム表示、一覧は空 |
| 検索実行中 | 検索ボタン無効化、ローディング表示 |
| 検索結果表示 | 該当会員を一覧表示 |
| 詳細表示 | 選択会員の情報を詳細表示 |
| 編集中 | 項目を編集可能に |
| 更新中 | 送信ボタン無効化 |
| 更新成功 | 成功メッセージ、一覧へ戻る |
| 更新失敗 | エラーメッセージ、情報保持 |

---

## 10. 操作フロー

```
START
  ↓
[検索条件入力（会員ID、施設タイプ等）]
  ↓
[検索ボタン押下] → GET /api/v1/caremembers → 一覧表示
  ↓
[会員選択] → GET /:careeMemberId → 詳細表示
  ↓
[編集または確認]
  ├→ [更新ボタン押下] → PATCH /:careeMemberId → 成功/失敗
  └→ [キャンセル] → 検索結果へ戻る
  ↓
END
```

---

## 16. 実装AI向け指示

**状態管理**:
- 検索条件、検索結果、選択会員、編集フォーム値を分離管理
- 状態に応じて編集可否を動的に制御

**API呼び出し順序**:
1. 検索実行 → GET /api/v1/caremembers
2. 会員選択 → GET /:careeMemberId
3. 更新ボタン → PATCH /:careeMemberId
4. 成功時は検索結果リストを再取得
