# UI仕様書: スケジュール確定画面

**画面日本語名**: スケジュール確定画面  
**画面英語名**: ScheduleConfirmationScreen  
**所属BUC**: SchedulePlanningFlow  

**目的**: 計画されたスケジュールを確認して確定判定を実行。確定内容をスタッフに通知。

---

## 表示/入力項目

| 項目名 | 型 | 必須 | 編集可否 |
|-------|---|-----|--------|
| スケジュールID | number | - | - |
| 会員名 | string | - | - |
| 確定日時 | datetime | ○ | ○ |
| 訪問先 | string | - | - |
| スタッフ | string | - | - |

---

## API利用

**確定**: POST /api/v1/schedules/:scheduleId/confirm

**リクエスト**:
```json
{
  "confirmedDateTime": "string (YYYY-MM-DD HH:MM)"
}
```

---

## 操作フロー

```
START → [計画スケジュール表示]
  → [確定判定を実行]
  → [確定ボタン押下] → POST /:id/confirm
    ├→ [成功] → 成功メッセージ
    └→ [失敗] → エラーメッセージ
  → END
```

---

## 機能要求

| 要求ID | 要求内容 | 優先度 |
|-------|--------|-------|
| FR-001 | スケジュールを確定できること | 必須 |
| FR-002 | 確定内容をスタッフに通知すること | 必須 |
