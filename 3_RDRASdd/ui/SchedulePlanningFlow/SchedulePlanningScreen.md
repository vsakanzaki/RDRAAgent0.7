# UI仕様書: スケジュール計画画面

**画面日本語名**: スケジュール計画画面  
**画面英語名**: SchedulePlanningScreen  
**所属BUC**: SchedulePlanningFlow  

**目的**: 介護会員の要望とスタッフのスキル・働き方を考慮した計画案を作成・編集。複数案を比較検討。

---

## 表示/入力項目

| 項目名 | 型 | 必須 | 編集可否 |
|-------|---|-----|--------|
| スケジュールID | number | - | - |
| 会員名 | string | - | - |
| 計画日時 | datetime | ○ | ○ |
| 訪問先 | string | ○ | ○ |
| スタッフ | string | ○ | ○ |

---

## API利用

**計画作成**: POST /api/v1/schedules/plan

**リクエスト**:
```json
{
  "scheduleId": "number",
  "plannedDateTime": "string",
  "visitLocation": "string",
  "staffId": "number"
}
```

---

## 操作フロー

```
START → [計画フォーム表示]
  → [計画日時、訪問先、スタッフを入力]
  → [計画ボタン押下] → POST /plan
    ├→ [成功] → 成功メッセージ
    └→ [失敗] → エラーメッセージ
  → END
```

---

## 機能要求

| 要求ID | 要求内容 | 優先度 |
|-------|--------|-------|
| FR-001 | 計画案を作成できること | 必須 |
| FR-002 | 複数案を比較できること | 推奨 |
