# UI仕様書: スキル入力画面

## 1. 概要
**画面日本語名**: スキル入力画面  
**画面英語名**: SkillInputScreen  
**所属BUC**: StaffSkillManagementFlow  

**目的**: サービススタッフが自身のスキル情報を詳細に入力・申告。複数のスキルを管理可能。

---

## 2. 利用アクターと役割
| アクター | 役割・責務 |
|---------|---------|
| サービススタッフ | 自身のスキル情報を入力・申告 |

---

## 3. 関連UC
- **UC-001: スタッフスキル情報を入力する**

---

## 4. 画面目的
スタッフが保有するスキル情報を申告。スキル種別、取得日、説明を入力し、「申告」状態で記録。

---

## 5. 表示/入力項目

| 項目名 | 型 | 必須 | 編集可否 |
|-------|---|-----|--------|
| スタッフID | number | - | - |
| スキル種別 | enum | ○ | ○ |
| 取得日 | date | ○ | ○ |
| 説明 | string | - | ○ |

---

## 6. API利用（UCエンドポイント）

**スキル申告**: POST /api/v1/staff-skills/register

**リクエスト**:
```json
{
  "staffId": "number",
  "skillType": "enum",
  "acquisitionDate": "string (YYYY-MM-DD)",
  "description": "string (optional)"
}
```

**レスポンス (201 Created)**:
```json
{
  "skillId": "number",
  "staffId": "number",
  "skillType": "string",
  "acquisitionDate": "string",
  "skillState": "申告",
  "createdAt": "datetime"
}
```

---

## 7. 画面状態

| 状態 | 説明 |
|-----|------|
| 初期表示 | スキル入力フォーム |
| 入力中 | ユーザーが情報を入力 |
| 検証エラー | バリデーション失敗時エラー表示 |
| 申告中 | ボタン無効化 |
| 申告成功 | 成功メッセージ、スキル一覧へ遷移 |
| 申告失敗 | エラーメッセージ表示 |

---

## 8. 操作フロー

```
START → [フォーム表示]
  → [スキル種別、取得日、説明を入力]
  → [申告ボタン押下]
  → POST /register
    ├→ [成功] → 成功メッセージ → スキル一覧へ
    └→ [失敗] → エラーメッセージ → 入力画面へ
  → [キャンセル] → 前画面へ
  → END
```

---

## 9. 機能要求

| 要求ID | 要求内容 | 優先度 |
|-------|--------|-------|
| FR-001 | スキル情報を入力できること | 必須 |
| FR-002 | 複数のスキルを申告できること | 必須 |
| FR-003 | 申告後、スキル確認画面へ遷移すること | 推奨 |

---

## 10. バリデーション

| 項目 | 検証ルール |
|-----|---------|
| スキル種別 | 列挙値のみ（介護助手、介護福祉士、看護師、その他） |
| 取得日 | YYYY-MM-DD、過去日付 |
| 説明 | 0-500文字 |

---

## 11. 監査/権限制御

**権限**: 「サービススタッフ」（ログイン済みユーザー自身のスキルのみ申告可能）

**監査ログ**: スキル申告 → staffId, skillType, acquisitionDate, タイムスタンプ
