# UI仕様書: スキル確認画面

## 1. 概要
**画面日本語名**: スキル確認画面  
**画面英語名**: SkillConfirmationScreen  
**所属BUC**: StaffSkillManagementFlow  

**目的**: スタッフ管理担当者がスタッフが申告したスキル情報を確認・検証。「適正」「要確認」「保留」のいずれかを判定。

---

## 2. 利用アクターと役割
| アクター | 役割・責務 |
|---------|---------|
| スタッフ管理担当者 | 申告されたスキル情報を確認・検証 |

---

## 3. 関連UC
- **UC-002: スタッフスキル情報を確認する**

---

## 4. 画面目的
申告状態のスキル情報を表示し、管理者が確認結果を記録。スキル状態を「確認」に遷移。

---

## 5. 表示/入力項目

| 項目名 | 型 | 必須 | 編集可否 |
|-------|---|-----|--------|
| スキルID | number | - | - |
| スタッフ名 | string | - | - |
| スキル種別 | enum | - | - |
| 取得日 | date | - | - |
| 確認結果 | enum | ○ | ○ |

---

## 6. API利用（UCエンドポイント）

**スキル確認**: PATCH /api/v1/staff-skills/:skillId/verify

**リクエスト**:
```json
{
  "verificationResult": "enum (適正|要確認|保留)",
  "verificationDate": "string (YYYY-MM-DD)"
}
```

**レスポンス (200 OK)**:
```json
{
  "skillId": "number",
  "skillType": "string",
  "skillState": "確認",
  "verificationResult": "string",
  "verifiedAt": "datetime"
}
```

---

## 7. 画面状態

| 状態 | 説明 |
|-----|------|
| 初期表示 | 申告スキル一覧表示 |
| スキル選択 | 詳細情報表示 |
| 確認入力 | 確認結果を選択 |
| 確認中 | ボタン無効化 |
| 確認成功 | 成功メッセージ、一覧へ戻る |
| 確認失敗 | エラーメッセージ |

---

## 8. 操作フロー

```
START → [申告スキル一覧表示]
  → [スキル選択] → GET /:skillId
  → [詳細情報表示]
  → [確認結果を選択]
  → [確認ボタン押下] → PATCH /:id/verify
    ├→ [成功] → 成功メッセージ
    └→ [失敗] → エラーメッセージ
  → END
```

---

| 要求ID | 要求内容 | 優先度 |
|-------|--------|-------|
| FR-001 | 申告スキルを一覧表示できること | 必須 |
| FR-002 | 確認結果を記録できること | 必須 |
| FR-003 | スキル状態を「確認」に遷移させること | 必須 |

---

## 10. バリデーション

| 項目 | 検証ルール |
|-----|---------|
