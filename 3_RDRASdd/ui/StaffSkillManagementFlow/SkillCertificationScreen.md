# UI仕様書: スキル認定画面

## 1. 概要
**画面日本語名**: スキル認定画面  
**画面英語名**: SkillCertificationScreen  
**所属BUC**: StaffSkillManagementFlow  

**目的**: 確認済みのスキル情報に対して認定判定を実行し、認定結果を記録。認定済みスキルのみスケジュール計画で利用可能。

---

## 2. 利用アクターと役割
| アクター | 役割・責務 |
|---------|---------|
| スタッフ管理担当者 | スキル認定判定を実行・記録 |

---

## 3. 関連UC
- **UC-003: スタッフスキル情報を認定する**

---

## 4. 画面目的
確認状態のスキルを表示し、認定判定（認定/保留/非認定）を記録。スキル状態を「認定」に遷移。

---

## 5. 表示/入力項目

| 項目名 | 型 | 必須 | 編集可否 |
|-------|---|-----|--------|
| スキルID | number | - | - |
| スタッフ名 | string | - | - |
| スキル種別 | enum | - | - |
| 認定結果 | enum | ○ | ○ |
| 認定日 | date | ○ | ○ |

---

## 6. API利用（UCエンドポイント）

**スキル認定**: PATCH /api/v1/staff-skills/:skillId/approve

**リクエスト**:
```json
{
  "approvalResult": "enum (認定|保留|非認定)",
  "approvalDate": "string (YYYY-MM-DD)"
}
```

**レスポンス (200 OK)**:
```json
{
  "skillId": "number",
  "skillType": "string",
  "skillState": "認定",
  "approvalResult": "string",
  "approvedAt": "datetime"
}
```

---

## 7. 画面状態

| 状態 | 説明 |
|-----|------|
| 初期表示 | 確認済みスキル一覧 |
| スキル選択 | 詳細情報表示 |
| 認定入力 | 認定結果を選択 |
| 認定中 | ボタン無効化 |
| 認定成功 | 成功メッセージ |
| 認定失敗 | エラーメッセージ |

---

## 8. 操作フロー

```
START → [確認済みスキル一覧表示]
  → [スキル選択] → 詳細表示
  → [認定結果、認定日を入力]
  → [認定ボタン押下] → PATCH /:id/approve
    ├→ [成功] → 成功メッセージ
    └→ [失敗] → エラーメッセージ
  → END
```

---

## 9. 機能要求

| 要求ID | 要求内容 | 優先度 |
|-------|--------|-------|
| FR-001 | 確認済みスキルを一覧表示できること | 必須 |
| FR-002 | 認定結果を記録できること | 必須 |
| FR-003 | スキル状態を「認定」に遷移させること | 必須 |
| FR-004 | 認定済みスキルのみスケジュール計画で利用可能にすること | 必須 |

---

## 10. バリデーション

| 項目 | 検証ルール |
|-----|---------|
| 認定結果 | 列挙値のみ |
| 認定日 | YYYY-MM-DD、本日以降 |
