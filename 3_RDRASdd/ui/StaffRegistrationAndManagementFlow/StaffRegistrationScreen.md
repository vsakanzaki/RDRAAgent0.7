# UI仕様書: スタッフ登録画面

## 1. 概要
**画面日本語名**: スタッフ登録画面  
**画面英語名**: StaffRegistrationScreen  
**所属BUC**: StaffRegistrationAndManagementFlow  

**目的**: 新規スタッフの基本情報を入力し、システムに登録。スタッフIDは自動採番。

---

## 2. 利用アクターと役割
| アクター | 役割・責務 |
|---------|---------|
| スタッフ管理担当者 | 新規スタッフ基本情報の登録 |

---

## 3. 関連UC
- **UC-002: スタッフ基本情報を登録する**

---

## 4. 画面目的
新規スタッフの名前、資格、所属事業所を入力し、登録。

---

## 5. 表示/入力項目

| 項目名 | 型 | 必須 | 編集可否 |
|-------|---|-----|--------|
| スタッフID | number | - | - |
| 名前 | string | ○ | ○ |
| 資格 | string | ○ | ○ |
| 事業所 | enum | ○ | ○ |

---

## 6. API利用（UCエンドポイント）

**登録**: POST /api/v1/staff/register

**リクエスト**:
```json
{
  "name": "string",
  "qualification": "string",
  "businessLocationId": "number"
}
```

**レスポンス (201 Created)**:
```json
{
  "staffId": "number",
  "name": "string",
  "qualification": "string",
  "businessLocationId": "number",
  "businessLocationName": "string",
  "createdAt": "datetime"
}
```

**エラーハンドリング**:
- **400**: 必須項目不足
- **409**: 事業所が存在しない

---

## 7. 画面状態

| 状態 | 説明 |
|-----|------|
| 初期表示 | 空のフォーム |
| 入力中 | ユーザーが入力 |
| 検証エラー | バリデーション失敗時エラー表示 |
| 登録中 | ボタン無効化 |
| 登録成功 | 成功メッセージ、管理画面へ遷移 |
| 登録失敗 | エラーメッセージ表示 |

---

## 8. 操作フロー

```
START → [フォーム表示]
  → [名前、資格、事業所を入力]
  → [登録ボタン押下]
  → POST /register
    ├→ [成功] → 成功メッセージ → 管理画面へ遷移
    └→ [失敗] → エラーメッセージ → 入力画面へ戻す
  → [キャンセル] → 前画面へ
  → END
```

---

## 9. 機能要求

| 要求ID | 要求内容 | 優先度 |
|-------|--------|-------|
| FR-001 | スタッフ基本情報を入力できること | 必須 |
| FR-002 | 事業所ドロップダウンを提供すること | 必須 |
| FR-003 | 登録後、詳細管理画面へ遷移すること | 必須 |

---

## 10. バリデーション

| 項目 | 検証ルール |
|-----|---------|
| 名前 | 1-100文字、非空 |
| 資格 | 1-100文字、非空 |
| 事業所 | 列挙値のみ |

---

## 11. 監査/権限制御

**権限**: 「スタッフ管理担当者」

**監査ログ**: 登録操作 → staffId, name, qualification, businessLocationId, タイムスタンプ
