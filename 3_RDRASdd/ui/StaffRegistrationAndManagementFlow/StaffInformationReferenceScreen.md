# UI仕様書: スタッフ情報参照画面

## 1. 概要
**画面日本語名**: スタッフ情報参照画面  
**画面英語名**: StaffInformationReferenceScreen  
**所属BUC**: StaffRegistrationAndManagementFlow  

**目的**: スケジュール計画担当者がスケジュール計画に必要なスタッフ情報を検索・参照。スタッフのスキルと働き方を確認。

---

## 2. 利用アクターと役割
| アクター | 役割・責務 |
|---------|---------|
| スケジュール計画担当者 | スタッフ情報を検索・参照してスケジュール計画に利用 |

---

## 3. 関連UC
- **UC-001: スタッフ基本情報を提供する**

---

## 4. 画面目的
スケジュール計画に必要なスタッフの基本情報、スキル、働き方を検索・表示。

---

## 5. 表示/入力項目

| 項目名 | 型 | 必須 | 編集可否 |
|-------|---|-----|--------|
| スタッフID | number | - | - |
| 名前 | string | - | - |
| 資格 | string | - | - |
| 事業所 | string | - | - |
| スキル一覧 | array | - | - |
| 働き方 | string | - | - |

---

## 6. API利用（UCエンドポイント）

**詳細取得**: GET /api/v1/staff/:staffId

**一覧取得**: GET /api/v1/staff?businessLocationId={id}&name={name}&page={page}&limit={limit}

**レスポンス**:
```json
{
  "staffId": "number",
  "name": "string",
  "qualification": "string",
  "businessLocationName": "string",
  "skills": [{"skillType": "string", "skillState": "string"}],
  "workingStyle": {"workingStyleType": "string", "workingStyleState": "string"}
}
```

---

## 7. 画面状態

| 状態 | 説明 |
|-----|------|
| 初期表示 | 検索フォーム |
| 検索結果 | スタッフ一覧表示 |
| 詳細表示 | 選択スタッフの詳細 |

---

## 8. 操作フロー

```
START → [検索条件入力] → [検索] → GET /api/v1/staff
  → [スタッフ一覧表示]
  → [スタッフ選択] → GET /:staffId
  → [詳細表示] → スキル・働き方表示
  → [戻る] → 一覧へ
  → END
```

---

## 9. 機能要求

| 要求ID | 要求内容 | 優先度 |
|-------|--------|-------|
| FR-001 | スタッフを名前で検索できること | 必須 |
| FR-002 | 事業所でフィルタリングできること | 推奨 |
| FR-003 | スキル一覧を表示できること | 必須 |
| FR-004 | 働き方情報を表示できること | 必須 |

|-----|---------|
