# UI仕様書: スタッフ情報管理画面

## 1. 概要
**画面日本語名**: スタッフ情報管理画面  
**画面英語名**: StaffInformationManagementScreen  
**所属BUC**: StaffRegistrationAndManagementFlow  

**目的**: 登録済みスタッフの基本情報を検索・表示し、名前、資格、所属事業所を編集・更新。変更履歴を管理。

---

## 2. 利用アクターと役割
| アクター | 役割・責務 |
|---------|---------|
| スタッフ管理担当者 | スタッフ情報の検索・編集・更新 |

---

## 3. 関連UC
- **UC-003: スタッフ基本情報を管理する**

---

## 4. 画面目的
スタッフを検索し、基本情報を表示・編集。スキルや働き方が既に設定されている場合は注意喚起。

---

## 5. 表示/入力項目

| 項目名 | 型 | 必須 | 編集可否 |
|-------|---|-----|--------|
| スタッフID | number | - | - |
| 名前 | string | ○ | ○ |
| 資格 | string | ○ | ○ |
| 事業所 | enum | ○ | ○ |

---

## 6. API利用（UCエンドポイント）

**一覧・検索**: GET /api/v1/staff?businessLocationId={id}&name={name}&page={page}&limit={limit}

**詳細取得**: GET /api/v1/staff/:staffId

**更新**: PATCH /api/v1/staff/:staffId

**リクエスト**:
```json
{
  "name": "string (optional)",
  "qualification": "string (optional)",
  "businessLocationId": "number (optional)"
}
```

**レスポンス (200 OK)**:
```json
{
  "staffId": "number",
  "name": "string",
  "qualification": "string",
  "businessLocationId": "number",
  "businessLocationName": "string",
  "updatedAt": "datetime"
}
```

---

## 7. 画面状態

| 状態 | 説明 |
|-----|------|
| 初期表示 | 検索フォーム |
| 検索結果 | スタッフ一覧表示 |
| 詳細表示 | スタッフ詳細情報編集可能で表示 |
| 注意表示 | スキル・働き方設定済みの場合は注意喚起 |
| 更新中 | ボタン無効化 |
| 更新成功 | 成功メッセージ |
| 更新失敗 | エラーメッセージ |

---

## 8. 操作フロー

```
START → [検索条件入力]
  → [検索] → GET /api/v1/staff
  → [スタッフ一覧表示]
  → [スタッフ選択] → GET /:staffId
  → [詳細情報表示・編集]
  → [更新ボタン押下]
  → PATCH /:staffId
    ├→ [成功] → 成功メッセージ
    └→ [失敗] → エラーメッセージ
  → [キャンセル] → 検索結果へ
  → END
```
