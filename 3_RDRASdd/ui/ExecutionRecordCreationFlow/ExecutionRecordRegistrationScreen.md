# UI仕様書: 実施記録登録画面

**画面日本語名**: 実施記録登録画面  
**画面英語名**: ExecutionRecordRegistrationScreen  
**所属BUC**: ExecutionRecordCreationFlow  

**目的**: 事務担当者がサービススタッフが作成した実施記録を確認・検証してシステムに登録。登録状況を管理。

---

## 表示/入力項目

| 項目名 | 型 | 必須 | 編集可否 |
|-------|---|-----|--------|
| 記録ID | number | - | - |
| スケジュールID | number | - | - |
| 会員名 | string | - | - |
| 実施日時 | datetime | - | - |
| 登録状況 | enum | ○ | ○ |

---

## API利用

**登録**: POST /api/v1/execution-records/:recordId/register

**リクエスト**:
```json
{
  "registrationStatus": "enum"
}
```

---

## 操作フロー

```
START → [実施記録一覧表示]
  → [記録選択] → 詳細表示
  → [登録状況を確認]
  → [登録ボタン押下] → POST /:id/register
    ├→ [成功] → 成功メッセージ
    └→ [失敗] → エラーメッセージ
  → END
```
