# phase4：状態
- 「0_RDRAZeroOne/phase2/状態.tsv」を読込状態遷移の意味を理解する
- 「0_RDRAZeroOne/phase3/業務.tsv」を読みUCが行っていることを理解する
- 「0_RDRAZeroOne/phase3/情報.tsv」を読みコンテキストと情報、状態モデルを理解する

# 背景理解
- 「RDRAKnowledge/1_RDRA/RDRA.md」を読込み理解してください
- 「RDRAKnowledge/1_RDRA/RDRASheet.md」の「状態」用Sheetを読込み理解してください
- 状態モデルの状態はUCが遷移させる

## 状態モデルの整備
- 「phase2/状態.tsv」をベースにコンテキストと遷移UCを追記し「phase4/状態.tsv」を出力する
- 「phase2/状態.tsv」を順次読み以下の処理を行う
  - 「phase3/情報.tsv」の状態モデルに対応する「状態.tsv」の状態モデルがあれば、その情報のコンテキストを状態モデルのコンテキストとする
  - 状態から遷移先状態への遷移を行うUCを「phase3/BUC.tsv」のUCに対応させ、BUCのUCを設定する
-「状態モデル」フォーマットで出力する

### コンテキスト設定例
#### phase3/情報
コンテキスト、情報、～、状態モデル、～
在庫管理、在庫、～、"在庫状態、出荷状態"、～

#### phase4/状態
コンテキスト、状態モデル、～
在庫管理、在庫状態、～
在庫管理、出荷状態、～
販売管理、販売状態、～      //情報に対応するものがないので「販売状態」に対応するコンテキストを導出

### 遷移UCの設定例
状態モデル、状態、遷移UC、遷移先状態
会員状態、start、、未会員
会員状態、未会員、会員を登録する、活動中
会員状態、活動中、会員を休会にする、休会中
会員状態、活動中、会員を退会する、未会員
会員状態、休会中、会員を退会する、未会員
会員状態、未会員、、end
会員状態、休会中、、end

## 出力ルール
- 出力には必ずヘッダー（カラム名）行を出力する
- 前後に「```tsv」「```」を付ける
- 「状態モデル」以外の説明を出力しない

### 「状態モデル」フォーマット
- ビジネスを駆動するために管理する必要のある状態モデルを表す
- ６カラムのタブ区切りで出力してください
```tsv
「コンテキスト、状態モデル、状態、遷移UC、遷移先状態、説明」
- コンテキスト：状態モデルに対応する情報と同じコンテキストになる
- 状態モデル：状態遷移の管理単位となる
- 状態：遷移する前の状態と最後の状態を表す。最後の状態の場合は遷移先状態は"end"になる
- 遷移UC：状態を遷移させるユースケースを表す
- 遷移先状態：遷移後の状態と最初の状態を表す。最初の状態の場合は状態は"start"になる
- 説明：状態モデルが必要な理由と管理する状態遷移の説明
```
