# phase4：条件

- 「phase2/条件.md」を深く理解する
- 「phase2/状態.md」を深く理解する

# 背景理解
- 「RDRAKnowledge/1_RDRA/RDRA.md」を読込み理解してください
- 「RDRAKnowledge/1_RDRA/RDRASheet.md」の「バリエーション」用のSheetを読込み理解してください
- Phase4は最終的なRDRASheetのBUCの形式にする
  
## 条件の整備
- 「phase2/条件.tsv」をベースにコンテキスト、バリエーション、状態モデルを追記して「phase4/条件.tsv」を出力する
- 「phase2/条件.tsv」の条件を順次読み以下の処理を行う
  - 「phase2/バリエーション.md」のバリエーションは「条件に対応するバリエーションと状態モデル」に従って処理する
  - 「phase2/状態.md」の状態モデルは「条件に対応するバリエーションと状態モデル」に従って処理する
-「条件」フォーマットで出力する

### 条件に対応するバリエーションと状態モデル
- 条件は計算式や判断条件に名前を付けたものか、表形式の縦横の組合せ形式のものに分かれる
- 条件が表形式の場合にバリエーションと状態モデルに関わる場合がある
  - 表形式の条件の縦軸か横軸に当てはまる以下のものを当てはめる
    - 状態モデルが当てはまる場合は、条件の状態モデルに対応する名前を設定する
    - バリエーションが当てはまる場合は、条件のバリエーションに対応する名前を設定する

## 出力ルール
- 出力には必ずヘッダー（カラム名）行を出力する
- 前後に「```tsv」「```」を付ける
- 「条件」以外の説明を出力しない

### 「条件」のフォーマット
システムに関わるビジネスルールを定義する
- ５カラムのタブ区切りで出力してください
```tsv
「コンテキスト、条件、条件の説明、バリエーション、状態モデル」
- コンテキスト：条件が所属するコンテキストを表す　条件に関わる情報のコンテキストを使う
- 条件：ビジネスルールの名前を表す
- 条件の説明：条件の具体的な内容
- バリエーション：条件の縦軸/横軸となるバリエーションを表す、「、」区切りで複数のバリエーションを一つの文字列で表す
- 状態モデル：条件の縦軸/横軸となる状態モデルを表す、「、」区切りで複数の状態モデルを一つの文字列で表す

```
