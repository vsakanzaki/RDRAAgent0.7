# 背景理解

- 「RDRAKnowledge/1_RDRA/RDRASheet.md」の「バリエーション」用のSheetを読込み理解してください
- Phase4は最終的なRDRASheetの形式にする
  
## バリエーションのコンテキストの設定
- 「phase2/バリエーション.tsv」をベースにコンテキストを追記して「phase4/バリエーション.tsv」を出力する
- 「phase2/バリエーション.tsv」を順次読み以下の処理を行う
  - 「phase3/情報.md」のバリエーションに対応するバリエーションがあれば、その情報のコンテキストをバリエーションのコンテキストとする
- 情報に対応するバリエーションがない場合はバリエーションの役割を分類する名前をコンテキストとする
-「バリエーション」フォーマットで出力する

## コンテキスト設定例
### phase3/情報
コンテキスト、情報、～、バリエーション、～
在庫管理、在庫、～、"在庫区分、在庫ランク"、～

### phase4/バリエーション
コンテキスト、バリエーション、～
在庫管理、在庫区分、～
在庫管理、在庫ランク、～
販売管理、販売種別、～      //情報に対応するものがないので「販売種別」に対応するコンテキストを導出

## 出力ルール
- 出力には必ずヘッダー（カラム名）行を出力する
- 前後に「```tsv」「```」を付ける
- 「バリエーション」フォーマット以外の説明を出力しない

### 「バリエーション」のフォーマット
ビジネス上で使用する区分や種別などを定義する
- ４カラムのタブ区切りで出力してください
```tsv
「コンテキスト、バリエーション、値、説明」
- コンテキスト：バリエーションが所属するコンテキストを表す
- バリエーション：管理すべき区分や種別を洗い出す。
- 値：バリエーション内の取り得る複数の値（「、」複数の値をカンマ区切りで一つの文字列として表現）
- 説明：バリエーションの役割や活用用途を説明する
```
