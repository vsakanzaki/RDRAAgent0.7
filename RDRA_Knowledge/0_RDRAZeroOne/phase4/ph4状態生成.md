# phase4：状態
- 「0_RDRAZeroOne/phase3/ph3BUC.tsv」を読みUCが行っていることと状態モデルの関係を理解する
- 「0_RDRAZeroOne/phase2/ph2状態.tsv」を読み状態モデルの役割を理解する
- 以後の「ph3BUC.tsv」「ph2状態.tsv」の記述は上記のファイルのパスを指す
- 上記の入力から状態モデルを再整理し「ph4状態.tsv」を出力する

# 背景理解
- 「RDRA_Knowledge/1_RDRA/RDRA.md」を読込み理解してください
- ここではRDRAモデルの中の状態モデルを整備する
- 状態モデルは状態が遷移する状態の集合となる
- UCは情報を操作して状態を遷移させる

## 状態モデルの整備
- 「ph3BUC.tsv」からUC、状態モデル、From状態、To状態を読み、状態を遷移するUCを把握する
- 「ph2状態.tsv」から状態モデルの役割を理解する
- 「ph2状態.tsv」と「ph3BUC.tsv」で同じ状態モデルで異なる状態がある場合は「ph3BUC.tsv」を優先する
- 状態には「最初の状態の値」と「最後の状態の値」がそれぞれ複数ある
  - 「最初の状態の値」の表現「状態：空フィールド、、遷移先状態：最初の状態」
  - 「最後の状態の値」の表現「状態：最後の状態、、遷移先状態：空フィールド」
    - 最後の状態とはその状態から他に遷移しない状態
- 状態と遷移先状態が同じ場合は遷移は起こっていないので出力しない（自己遷移禁止）
- 「状態モデル」フォーマットの形式で出力する

## 重要：自己遷移の禁止
- 2列目「状態」と4列目「遷移先状態」が同じ値の行は絶対に出力しない。
- 同じ状態に留まることは「遷移」ではないため、状態遷移モデルとして意味がない。
- 例：「会員アクティブ → 会員情報を更新する → 会員アクティブ」のような行は出力禁止。
- 出力前に全行をチェックし、状態＝遷移先状態となっている行があれば削除すること。

## 出力ルール
- 出力ファイル：「0_RDRAZeroOne/phase4/ph4状態.tsv」
- 出力には必ずヘッダー（カラム名）行を出力する
- 空フィールドにするときは、タブ区切りを省略しない
- プログラムは絶対に生成しない

### 「状態モデル」フォーマット
出力形式：TSV（タブ区切り）
カラム構成：
1列目: 状態モデル：「ph3BUC.tsv」の状態モデルカラムの値を設定
2列目: 状態：「ph3BUC.tsv」のFrom状態を設定
3列目: 遷移UC：「ph3BUC.tsv」のUCカラムの値を設定
4列目: 遷移先状態：「ph3BUC.tsv」のTo状態を設定
5列目: 説明：状態遷移の意味をなぜその状態遷移が必要なのかを説明する
ヘッダー行を含め、各フィールドはタブ文字で区切ってください

## 重要：状態モデル取り違え防止ルール
- 1列目「状態モデル」は、必ず「ph3BUC.tsv」の「状態モデル」カラムの値をそのまま設定する。
- 出力行ごとに「遷移UC」をキーに「ph3BUC.tsv」の該当行を特定し、その行の「状態モデル」と出力1列目が完全一致すること。

## 重要：状態値の制約
- 状態/遷移先状態に「、」で複数状態を並べてはいけない（複数必要なら行を分割する）。
- メモ目的の行（遷移UC・遷移先状態が空で説明だけの行）は出力しない。

## 出力チェックリスト（強化）
- [ ] 遷移UCは「ph3BUC.tsv」のUCに存在する
- [ ] 各行が5カラム（4個のタブ区切り）である
- [ ] 各状態モデルについて、「最初の状態の値」の行が存在する
- [ ] 各状態モデルについて、「最後の状態の値」（他に遷移しない終端状態）の行が存在する
