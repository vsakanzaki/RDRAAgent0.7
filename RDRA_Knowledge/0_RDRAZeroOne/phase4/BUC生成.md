- 「0_RDRAZeroOne/phase3/UC条件.tsv」を読みUCにつながる条件を理解する
- 「0_RDRAZeroOne/phase3/UCアクター.tsv」を読みUCにつながるアクターを理解する
- 「0_RDRAZeroOne/phase3/UC外部システム.tsv」を読みUCにつながる外部システムを理解する
- 「0_RDRAZeroOne/phase3/UCタイマー.tsv」を読みUCにつながるタイマーを理解する
- 「BUC.tsv」、「UC条件.tsv」、「UCアクター.tsv」、「UC外部システム.tsv」、「UCタイマー.tsv」の記述は上記のファイルのパスを指す
- 上記ファイルから「phase4/BUC.tsv」を出力する

# 背景理解
- 「RDRA_Knowledge/1_RDRA/RDRA.md」を読込みRDRAの構造を理解してください
- 「phase2/業務」をベースに「phase4/BUC」を作成する
- ユースケースにつながる情報を操作しシステムの一貫性を維持する  
- ユースケースが外部のシステムと連携する場合は外部システムと紐づける
- ユースケースが従うビジネスルールがある場合は条件を紐づける
- ユースケースはアクターが画面を使って操作する
- ユースケースが決まった日次に実行するときはタイマーを紐づける
- 「BUC.tsv」の「状態モデル」カラムは使用しない

## phase2/BUCからphase4/BUCの出力
- 「BUC.tsv」と「phase4/BUC」で同じ項目にはそのまま出力する

## UCに関わるものを関連オブジェクトとしてまとめる
- 「phase2/BUC」の説明を参考に「UC」につながる情報、条件、アクター、外部システム、タイマーを「BUC.tsv」に関連オブジェクトとして追記する
- 「BUC.tsv」のUCをベースに以下の各ファイルの内容をUCで結びつける
  - 「BUC.tsv」の情報     ⇒ 関連モデル1:"情報"  / 関連オブジェクト1:「BUC.tsv」の情報 / 関連モデル2: / 関連オブジェクト2:       
  - 「UC条件.tsv」       ⇒ 関連モデル1:"条件"  / 関連オブジェクト1:「UC条件.tsv」の条件 / 関連モデル2: / 関連オブジェクト2:     
  - 「UCアクター.tsv」    ⇒ 関連モデル1:"画面"  / 関連オブジェクト1:「UCアクター.tsv」の画面 / 関連モデル2:"アクター" / 関連オブジェクト2:「UCアクター.tsv」のアクター
  - 「UC外部システム.tsv」⇒ 関連モデル1:"イベント"  / 関連オブジェクト1:「UC外部システム.tsv」のイベント / 関連モデル2:"外部システム" / 関連オブジェクト2:「UC外部システム.tsv」の外部システム
  - 「UCタイマー.tsv」    ⇒ 関連モデル1:"タイマー"  / 関連オブジェクト1:「UCタイマー.tsv」のタイマー / 関連モデル2: / 関連オブジェクト2:
- 「BUC」のフォーマットで出力する

## 出力ルール
- 出力ファイル：「0_RDRAZeroOne/phase4/BUC.tsv」
- 出力には必ずヘッダー（カラム名）行を出力する
- 空フィールドにするときは、タブ区切りを省略しない
- プログラムは絶対に生成しない。

### 「BUC」のフォーマット
出力形式：TSV（タブ区切り）
カラム構成：
1列目: 業務：業務を分類する名前　
2列目: BUC：洗い出した「BUC」の名前。
3列目: 先：空フィールドにする（タブ区切りは省略しない）
4列目: アクティビティ：業務フロー内の各アクティビティを表す
5列目: 次：空フィールドか「↓」をセットする
6列目: UC：「phase2/BUC.tsv」のUCをセットする
7列目: 関連モデル1：次の候補[情報,条件,画面,イベント,タイマー]　この中の一つに決まる
8列目: 関連オブジェクト1：関連モデル1のモデルに対応するオブジェクト
9列目: 関連モデル2：次の候補[アクター,外部システム]　この中の一つまたは空フィールドにする
10列目: 関連オブジェクト2：関連モデル2のモデル[[アクター,外部システム]に対応するオブジェクト、関連モデル2が空フィールドの時は空フィールドにする
11列目: 説明：UCの説明
12列目: 入出力要求：関連モデル1が「画面」の時に、画面を開発するときの要求を設定する。それ以外は空フィールドにする
- ヘッダー行を含め、各フィールドはタブ文字で区切ってください
- 空フィールドの場合は「、」だけを出力しない

#### 「phase2/BUC.tsv」「UC条件.tsv」「UCアクター.tsv」「UC外部システム.tsv」「UCタイマー.tsv」から「phase4/BUC」への変換ルール
##### 変換ルール：
- 「業務」,「BUC」,「アクティビティ」,「UC」,「説明」カラムの値はそのままコピーする
- 「先」空フィールドにする（タブ区切りは省略しない）
- それ以外のカラムは以下のルールに従う
- カラム内の値に「、」か「,」がある場合は値を「、」か「,」で分割し、分割した数に応じて複数行で出力する
- 情報：「関連モデル1」を「情報」にし、「関連オブジェクト1」に「phase2/BUC.tsv」の「情報」カラムの値をセットし、「関連モデル2」と「関連オブジェクト2」には空フィールドにする
- 条件」：「関連モデル1」を「条件」にし、「関連オブジェクト1」に「UC条件.tsv」の「条件」カラムの値をセットし、「関連モデル2」と「関連オブジェクト2」には空フィールドにする
- アクター：「関連モデル1」を「画面」にし、「関連オブジェクト1」に「UCアクター.tsv」の「画面」カラムをセット、「関連モデル2」を「アクター」にし、「関連オブジェクト2」に「UCアクター.tsv」の「アクター」カラムの値をセットする
- 外部システム：「関連モデル1」を「イベント」にし、「関連オブジェクト1」に「イベント名」をセット、「関連モデル2」を「外部システム」にし、「関連オブジェクト2」に「UC外部システム.tsv」の「外部システム」カラムの値をセットする
- タイマー：「関連モデル1」を「タイマー」にし、「関連オブジェクト1」に「UCタイマー.tsv」の「タイマー」カラムの値をセットし、「関連モデル2」と「関連オブジェクト2」には空フィールドにする
