# phase4：BUC
- 「0_RDRAZeroOne/phase3/BUC.tsv」を読みUCにつながる価値受益アクター、価値提供アクター、情報、条件、外部システムを理解し、再整理し「BUC.tsv」を出力する

# 背景理解
- 「RDRA_Knowledge/1_RDRA/RDRA.md」を読込みRDRAの構造を理解してください
- phase1のアクターと業務は初期要望から個別に洗い出している
- 「phase2/業務」をベースに「phase3/BUC」を作成する
- システムはユースケースにつながる情報を操作しシステムの一貫性を維持する  
- アクターはユースケースにつながる情報を操作しシステムの一貫性を維持する
- システムが外部システムと連携する場合はユースケースと連携する
- ユースケースが従うビジネスルールがある場合は条件を紐づける

## UCに関わる操作するを洗出す
- 「phase3/業務」の項目をそのまま「phase4/BUC」に出力する
- 「phase3/業務」の説明を参考に「UC」につながる情報、条件、外部システムを追記する
  - アクティビティが行う仕事をシステムが行う場合はユースケース（UC）に情報、条件、外部システム、アクターとの連携で記述する
  - UCで行うことをシステムで整合させるために情報とつなげる
  - システムが外部のシステムと連携する場合に外部システムをUCにつなげる
  - システムがビジネスルールに従うときに条件とつなげる
- 「BUC」のフォーマットで出力する

## 出力ルール
- 出力ファイル：「0_RDRAZeroOne/phase4/BUC.tsv」
- 出力には必ずヘッダー（カラム名）行を出力する

### 「BUC」のフォーマット
出力形式：TSV（タブ区切り）
カラム構成：
1列目: 業務：業務を分類する名前　
2列目: BUC：洗い出した「BUC」の名前。
3列目: 先：""をセットする
4列目: アクティビティ：業務フロー内の各アクティビティを表す
5列目: 次：""をセットする
6列目: UC：「phase3/BUC.tsv」のUCをセットする
7列目: 関連モデル1：次の候補[情報,条件,画面,イベント,タイマー]　この中の一つに決まる
8列目: 関連オブジェクト1：関連モデル1のモデルに対応するオブジェクト
9列目: 関連モデル2：次の候補[アクター,外部システム,""]　この中の一つに決まる
10列目: 関連オブジェクト2：関連モデル2のモデル[[アクター,外部システム]に対応するオブジェクト、関連モデル2が""の時は""をセット
11列目: 説明：「ユースケース（UC）」の説明
ヘッダー行を含め、各フィールドはタブ文字で区切ってください。

#### 「phase3/BUC」から「phase4/BUC」への変換ルール
##### 変換ルール：
- 「業務」,「BUC」,「アクティビティ」,「ユースケース」,「説明」カラムの値はそのままコピーする
- 「先」「次」カラムには""をセットする
- それ以外のカラムは以下のルールに従う
- カラム内の値に「、」か「,」がある場合は値を「、」か「,」で分割し、分割した数に応じて複数行で出力する
- 「情報」カラムは「関連モデル1」を「情報」にし、「関連オブジェクト1」に「情報」カラムの値をセットし、「関連モデル2」と「関連オブジェクト2」にはブランクをセットする
- 「条件」カラムは「関連モデル1」を条件にし、「関連オブジェクト1」に「条件」カラムの値をセットし、「関連モデル2」と「関連オブジェクト2」にはブランクをセットする
- 「アクターカラム」は「関連モデル1」を「画面」にし、「関連オブジェクト1」に「画面名」をセット、「関連モデル2」を「アクター」にし、「関連オブジェクト2」に「アクター」カラムの値をセットする
- 「外部システム」カラムは「関連モデル1」を「イベント」にし、「関連オブジェクト1」に「イベント名」をセット、「関連モデル2」を「外部システム」にし、「関連オブジェクト2」に「外部システム」カラムの値をセットする
- 「画面名」は「UC」が提供する画面として適切な名前にする
- 「イベント名」は「UC」と「外部システム」との連携として適切な名前にする

##### 変換例：
###### phase3のBUCファイル形式：
- 業務A/BUC_A/アクターA/アクターB,アクターC/アクティビティ_A/UC_A/情報A,情報B/条件A,条件B/外部システムA/説明A

###### phase4のBUC形式：
- 業務A/BUC_A/""/アクティビティ_A/""/UC_A/情報/情報A/""/""/説明A
- 業務A/BUC_A/""/アクティビティ_A/""/UC_A/情報/情報B/""/""/""
- 業務A/BUC_A/""/アクティビティ_A/""/UC_A/条件/条件A/""/""/""
- 業務A/BUC_A/""/アクティビティ_A/""/UC_A/条件/条件B/""/""/""
- 業務A/BUC_A/""/アクティビティ_A/""/UC_A/画面/画面A/アクター/アクターA/""
- 業務A/BUC_A/""/アクティビティ_A/""/UC_A/イベント/イベントA/外部システム/外部システムA/""

###### 変換例：phase3の「価値受益アクター、価値提供アクター、情報、条件、アクター、外部システム」カラムからphase4の「関連モデル1、関連オブジェクト1、関連モデル2、関連オブジェクト2」への変換
- phase3 -> phase4
- 価値受益アクター：アクターA -> 関連モデル1：画面,関連オブジェクト1：画面A,関連モデル2：アクター,関連オブジェクト2：アクターA
- 価値提供アクター：アクターB -> 関連モデル1：画面,関連オブジェクト1：画面B,関連モデル2：アクター,関連オブジェクト2：アクターB
- 価値提供アクター：アクターC -> 関連モデル1：画面,関連オブジェクト1：画面C,関連モデル2：アクター,関連オブジェクト2：アクターC
- 情報：情報A -> 関連モデル1：情報,関連オブジェクト1：情報A,関連モデル2："",関連オブジェクト2：""
- 条件：条件A -> 関連モデル1：条件,関連オブジェクト1：条件A,関連モデル2："",関連オブジェクト2：""
- 外部システム：外部システムA -> 関連モデル1：イベント,関連オブジェクト1：イベントA,関連モデル2："外部システム",関連オブジェクト2："外部システムA"
