# 背景理解
- 「RDRAKnowledge/1_RDRA/RDRA.md」を読込みRDRAモデルの構造を理解してください
- ここではRDRAモデルの中の情報を整備する
  
## 情報の整理
- 「phase2/情報.tsv」をベースにコンテキストで分類し、状態モデル、バリエーションと紐付けて「phase3/情報.tsv」に出力する
- 情報を役割に応じてコンテキストとして分類する
- コンテキストは情報を中心とした管理単位となる
- コンテキストはドメインのグループとして活用する
- 「phase2/情報.tsv」を順次読込以下のことを行う
  - 情報に関係する状態モデルを関連付ける
  - 情報はユニークなIDを持ち、状態モデルもユニークなIDの状態遷移を表す
  - 同じIDをもつものを関連付ける
  - 情報に従属するバリエーションを関連付ける
    - 例：会員ランク（バリエーション）は会員（情報）に従属する
  - 初期要望分析にUCと情報、条件（ビジネスルール）、外部システムの関係の記述があれば、それを含める
- 上記の操作を行い「情報」のフォーマットで出力する

## 出力ルール
- 出力には必ずヘッダー（カラム名）行を出力する
- 前後に「```tsv」「```」を付ける
- 「情報」以外の説明を出力しない

### 「情報」のフォーマット
BUCを進めるうえで必要な情報を元にシステムで扱う情報を表す
- ７カラムのタブ区切りで出力する
```tsv
「コンテキスト、情報、属性、関連情報、状態モデル、バリエーション、説明」
- コンテキスト：情報を役割に応じてグルーピングする。名前の後ろに「～管理」を付ける
- 情報：ユニークなIDで一意に識別できるデータを表す
- 属性：情報が保持する複数の属性を表す。属性は「、」区切りで一つの文字列として保持する
- 関連情報：情報同士の関連を表す。情報と関連する情報を「、」区切りで一つの文字列として保持する
- 状態モデル：状態で管理する状態モデルを表します、「、」区切りで複数の状態モデルを一つの文字列で表す
- バリエーション：情報に従属するバリエーションを表します、「、」区切りで複数のバリエーションを一つの文字列で表す
- 説明：情報が必要な理由を説明
```
