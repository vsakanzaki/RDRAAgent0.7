# 背景理解
- 「RDRA_Knowledge/1_RDRA/RDRA.md」を読込み理解してください
- ここではRDRAモデルの中の「バリエーション」を洗い出す
  
## バリエーションの洗出し
- ビジネスパラメータからビジネスとして管理すべき区分や種別を洗い出す。
- バリエーションはビジネス上の区分や種類を表す
-「バリエーション」フォーマットで出力する

## 実行ルール
- 出力内容について判断に迷った場合は可能性の一番高いものを選択する

## 出力ルール
- 出力には必ずヘッダー（カラム名）行を出力する
- 空フィールドにするときは、タブ区切りを省略しない
- プログラムは絶対に生成しない

### 「バリエーション」のフォーマット
出力先：0_RDRAZeroOne/phase3/ph3バリエーション.tsv
出力形式：TSV（タブ区切り）
カラム構成：
1列目: バリエーション：管理すべき区分や種別を洗い出す。
2列目: 値：バリエーション内の取り得る複数の値（複数ある時は「、」で区切る）
3列目: 説明：バリエーションの役割や活用用途を説明する
- ヘッダー行を含め、各フィールドはタブ文字で区切ってください。

## 出力チェックリスト
- [ ] 各行が3カラム（2個のタブ区切り）であることを確認する
