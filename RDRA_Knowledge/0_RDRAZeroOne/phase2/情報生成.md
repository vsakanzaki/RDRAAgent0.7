# phase2：情報
- 「0_RDRAZeroOne/phase1/初期要望分析.md」を読み初期の要望を理解する
- 「0_RDRAZeroOne/phase1/業務.tsv」を読み初期要望の業務を理解する
- 「0_RDRAZeroOne/phase1/情報.tsv」を読み初期要望の情報を理解する

# 背景理解
- 「RDRAKnowledge/1_RDRA/RDRA.md」を読込みRDRAモデルの構造を理解してください
- ここではRDRAモデルの中の情報を整備する
- 業務/BUCは情報を使って様々な処理を実現する
  
## 情報の洗出し
- 「phase1/情報.tsv」をベースに「phase2/情報.tsv」を出力する
- 「phase1/業務.tsv」を理解し、BUCの中で利用する情報を洗い出す
- BUCから洗い出した情報が「phase1/情報.tsv」に存在しないときは、「phase2/情報.tsv」に情報を追加する
- 「phase1/情報.tsv」に属性と関連情報を追加する
- 情報が持つべき属性を洗い出す
- 情報同士の関連を関連情報として洗い出す
- 「初期要望分析.md」にデータの属性に関わる記述があれば属性として追記する
- 「初期要望分析.md」にデータ同士の関連に関わる記述があれば関連情報として追記する
-「情報」フォーマットで出力する

## 関連情報の例
- 会員と個人情報、会員と購買記録の関係が記述されている

### phase2/情報.tsv
情報、～、関連情報
------------------
会員、～、"個人情報、購買記録"
個人情報、～、""
購買記録、～、""

## 出力ルール
- 出力には必ずヘッダー（カラム名）行を出力する
- 前後に「```tsv」「```」を付ける
- 「情報」以外の説明を出力しない

### 「情報」のフォーマット
BUCを進めるうえで必要な情報を元にシステムで扱う情報を表す
- ４カラムのタブ区切りで出力する
```tsv
「情報、属性、関連情報、説明」
- 情報：ユニークなIDで一意に識別できるデータを表す
- 属性：情報が保持する複数の属性を表す。属性は「、」区切りで一つの文字列として保持する
- 関連情報：情報同士の関連を表す。情報と関連する情報を「、」区切りで一つの文字列として保持する
- 説明：情報が必要な理由を説明
```
