# phase1：ビジネスポリシー
- 「初期要望.txt」から情報を洗い出す

# 背景理解
- 初期要望は実現したいビジネスとそれを実現するシステムについて記述している
- ビジネスとシステムを深堀する時の様々な判断基準とする

## ビジネスポリシーの洗出し
- 初期要望からビジネスに関するポリシーを洗い出してください
- ビジネスポリシーが記述していない場合は記述されていることから、必要となるビジネスポリシーを類推する
-「ビジネスポリシー」フォーマットで出力する

## 出力ルール
- 出力には必ずヘッダー（カラム名）行を出力する
- 前後に「```tsv」「```」を付ける
- 「ビジネスポリシー」以外の説明を出力しない

### 「ビジネスポリシー」のフォーマット
ビジネスを進めるうえで必要なポリシーを表す
- ２カラムのタブ区切りで出力する
```tsv
「ビジネスポリシー、説明」
- ビジネスポリシー：ビジネスを行う上での方針を表す
- 説明：ビジネスポリシーが必要な背景を説明する
```
