# phase1：要求
- 「初期要望.txt」から機能要求と非機能要求を洗い出し「要求.tsv」を出力

# 背景理解
- 初期要望は実現したいビジネスとそれを実現するシステムについて記述している
  
## 要求の洗出し
- 初期要望から機能要求、非機能要求に関わるものを洗い出す
- 要求は「～したい」「～すること」「～できる」などで表現される
- 機能要求：具体的なビジネス、システムの機能にたいする要求を対象にする
- 非機能要求：使いやすさや、レスポンス時間など、具体的な機能ではない要求を指す
- 初期要望の中に処理を決まったタイミングで実行することが記述されていれば、定時実行要求として洗い出す
  - 例：「毎週末に実行」「月末にまとめる」「毎日実行する」

## 実行ルール
- 初期要望から要求が導出できないときはヘッダーだけをファイルに出力する
- 出力内容について判断に迷った場合は可能性の一番高いものを選択する

## 出力ルール
- 出力ファイル：「0_RDRAZeroOne/phase1/要求.tsv」
- 出力には必ずヘッダー（カラム名）行を出力する
- 空フィールドにするときは、タブ区切りを省略しない
- プログラムは絶対に生成しない

### 「要求」出力フォーマット
出力形式：TSV（タブ区切り）
カラム構成：
1列目: 要求：初期要望に具体的に記述されている要求
2列目: アクター：要求を出しているアクターが明示されている場合はアクターを設定。記述なしの場合は空フィールドにする
3列目: 機能非機能：機能か非機能かを設定　　「機能 or 非機能」
- ヘッダー行を含め、各フィールドはタブ文字で区切ってください。
- 空フィールドの場合は「、」だけを出力しない

## 出力チェックリスト
- [ ] 各行が3カラム（2個のタブ区切り）であることを確認する
