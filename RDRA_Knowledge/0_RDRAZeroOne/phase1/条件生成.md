# phase1：条件
- 「初期要望.txt」から条件を洗い出し「条件.tsv」を出力する

# 背景理解
- 「RDRA_Knowledge/1_RDRA/RDRA.md」を読込み理解してください
- 初期要望は実現したいビジネスとそれを実現するシステムについて記述している
- ここではRDRAモデルの中の「条件」を洗い出す
  
## 条件の洗出し
- 初期要望からビジネスとして必要なビジネスルールを洗い出す。
- ビジネスルールが必要な場面と理由を利用場面として洗い出す
- ビジネスルールを条件として洗い出す
-「条件」フォーマットで出力する

## 実行ルール
- 初期要望から業務とBUCが導出できないときはヘッダーだけをファイルに出力する
- 出力内容について判断に迷った場合は可能性の一番高いものを選択する

## 出力ルール
- 出力ファイル：「0_RDRAZeroOne/phase1/条件.tsv」
- 出力には必ずヘッダー（カラム名）行を出力する
- 空フィールドにするときは、タブ区切りを省略しない
- プログラムは絶対に生成しない。

### 「条件」のフォーマット
出力形式：TSV（タブ区切り）
カラム構成：
1列目: 条件：条件はビジネスルールを表す
2列目: 条件の説明：必要なルール（条件の具体的な内容）を条件の説明として設定
3列目: 利用場面：条件の役割や活用用途を説明する
ヘッダー行を含め、各フィールドはタブ文字で区切ってください。

## 出力チェックリスト
- [ ] 各行が3カラム（2個のタブ区切り）であることを確認する
