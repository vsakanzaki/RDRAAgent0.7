# phase1：状態
- 「初期要望.txt」から状態モデルを洗い出す

# 背景理解
- 「RDRAKnowledge/1_RDRA/RDRA.md」を読込み理解してください
- 初期要望は実現したいビジネスとそれを実現するシステムについて記述している
- ここではRDRAモデルの中の状態モデルを洗い出す
- 状態モデルは状態が遷移する状態の集合となる
  
## 状態モデルの洗出し
- 初期要望からビジネスとして管理すべき状態モデルを洗い出す。
- 状態モデル毎に管理すべき状態を洗い出す
- 状態モデルは必ず、複数の状態を持つ
- ビジネスは状態を使って今の状況を把握し、次の状態に遷移させる
-「状態モデル」フォーマットで出力する

## 出力ルール
- 出力には必ずヘッダー（カラム名）行を出力する
- 前後に「```tsv」「```」を付ける
- 「状態モデル」以外の説明を出力しない

### 「状態モデル」フォーマット
- ビジネスを駆動するために管理する必要のある状態モデルを表す
- ３カラムのタブ区切りで出力してください
```tsv
「状態モデル、状態、説明」
- 状態モデル：状態遷移の管理単位となる
- 状態：状態モデルがとる複数の状態を洗い出す。「、」区切りで一つの文字列として洗い出す
- 説明：状態モデルが必要な理由と管理する状態を説明
```
