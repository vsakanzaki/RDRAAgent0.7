# phase1：状態
- 「初期要望.txt」から状態モデルを洗い出し「状態.tsv」を出力する

# 背景理解
- 「RDRA_Knowledge/1_RDRA/RDRA.md」を読込み理解してください
- 初期要望は実現したいビジネスとそれを実現するシステムについて記述している
- ここではRDRAモデルの中の状態モデルを洗い出す
- 状態モデルは状態が遷移する状態の集合となる
  
## 状態モデルの洗出し
- 初期要望からビジネスとして管理すべき状態モデルを洗い出す
- 状態モデル毎に管理すべき状態を洗い出す
- 状態モデルは必ず、複数の状態を持つ
- ビジネスは状態を使って今の状況を把握し、次の状態に遷移させる
-「状態モデル」出力フォーマットで出力する

## 実行ルール
- 初期要望から業務とBUCが導出できないときはヘッダーだけをファイルに出力する
- 出力内容について判断に迷った場合は可能性の一番高いものを選択する
- 空フィールドにするときは、タブ区切りを省略しない

## 出力ルール
- 出力ファイル：「0_RDRAZeroOne/phase1/状態.tsv」
- 出力には必ずヘッダー（カラム名）行を出力する
- 空フィールドにするときは、タブ区切りを省略しない
- プログラムは絶対に生成しない

### 「状態モデル」出力フォーマット
出力形式：TSV（タブ区切り）
カラム構成：
1列目: 状態モデル：状態遷移の管理単位となる
2列目: 状態：状態モデルがとる複数の状態を洗い出す。「、」区切りで一つの文字列として洗い出す
3列目: 説明：状態モデルが必要な理由と管理する状態を説明
ヘッダー行を含め、各フィールドはタブ文字で区切ってください

## 出力チェックリスト
- [ ] 各行が3カラム（2個のタブ区切り）であることを確認する
