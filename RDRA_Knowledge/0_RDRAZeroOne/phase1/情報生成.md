# phase1：情報
- 「初期要望.txt」から情報を洗い出す

# 背景理解
- 「RDRAKnowledge/1_RDRA/RDRA.md」を読込み理解してください
- 初期要望は実現したいビジネスとそれを実現するシステムについて記述している
- ここではRDRAモデルの中の情報を洗い出す
- ビジネスは情報を使って様々な処理を実現する
  
## 情報の洗出し
- 初期要望からビジネスとして管理すべき情報を洗い出す。
- 情報は必ずユニークなIDを持ち、一意に識別できる
- ビジネスは情報に状態を保持する
- 「初期要望.txt」にデータに関わる記述がある場合は、情報として洗い出す
-「情報」フォーマットで出力する

## 出力ルール
- 出力には必ずヘッダー（カラム名）行を出力する
- 前後に「```tsv」「```」を付ける
- 「情報」以外の説明を出力しない

### 「情報」のフォーマット
BUCを進めるうえで必要な情報を元にシステムで扱う情報を表す
- ２カラムのタブ区切りで出力する
```tsv
「情報、説明」
- 情報：ユニークなIDで一意に識別できるデータを表す
- 説明：情報が必要な理由を説明
```
