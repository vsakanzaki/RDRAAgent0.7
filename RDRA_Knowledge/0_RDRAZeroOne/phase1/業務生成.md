# phase1：業務
- 「初期要望.txt」から業務、業務フローの単位を洗い出す

# 背景理解
- 「RDRAKnowledge/1_RDRA/RDRA.md」を読込み理解してください
- 初期要望は実現したいビジネスとそれを実現するシステムについて記述している
- ここではRDRAモデルの中の「業務」を洗い出す
  
## 業務とBUCの洗出し
- 初期要望からビジネスとして実現すべき業務とその中の業務フローとなるBUCを洗い出す。
- 業務は「販売管理」「在庫管理」などのように企業の中の大きな管理単位となる
- 業務は１つ以上のBUCを含み、階層構造になる
- BUCは業務フローの単位となる
- BUCは価値受益アクターに価値を提供する
- BUCは複数の価値提供アクターによって価値を提供する
- BUCが価値を提供するのは一人の価値受益アクターでなる
-「業務」フォーマットで出力する

## 出力ルール
- 出力には必ずヘッダー（カラム名）行を出力する
- 前後に「```tsv」「```」を付ける
- 「業務」以外の説明を出力しない

### 「業務」のフォーマット
- システム化対象の業務を定義する
- ５カラムのタブ区切りで出力してください
```tsv
「業務、BUC、価値受益アクター、価値提供アクター、説明」
- 業務：業務を分類する名前　例：「販売管理」「在庫管理」など
- BUC：洗い出した「BUC」の名前。名前の後ろに「～フロー」をつける
- 価値受益アクター：BUCから価値を得る一つのアクター
- 価値提供アクター：BUCの価値を提供するために関わる複数のアクター
- 説明：BUCの内容を説明
```
