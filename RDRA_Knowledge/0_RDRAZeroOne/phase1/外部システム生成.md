# phase1：外部システム
- 「初期要望.txt」から外部システムを洗い出す

# 背景理解
- 「RDRAKnowledge/1_RDRA/RDRA.md」を読込み理解してください
- 初期要望は実現したいビジネスとそれを実現するシステムについて記述している
- ここではRDRAモデルの中の「外部システム」を洗い出す
- 「外部システム」は業務を実現するために連携する外部のシステムを表す
  
## 外部システムの洗出し
- 初期要望からシステムとして必要な外部システムを洗い出す。
- 「業務」「システム」に関わる「外部システム」（システムに関わる外部のシステムの役割）を見つけ、「外部システム」のフォーマットで出力する
- 外部システムの役割に応じて幾つかの分類に分ける

## 出力ルール
- 出力には必ずヘッダー（カラム名）行を出力する
- 前後に「```tsv」「```」を付ける
- 「外部システム」以外の説明を出力しない

### 「外部システム」のフォーマット
システムに関わる外部のシステムを定義する
- ３カラムのタブ区切りで出力してください
```tsv
「外部システム群、外部システム、役割」
- 外部システム群：外部システムの役割に応じて分類する
- 外部システム：システムに関わる外部のシステム。外部システム名は役割が分かる名前にする
- 役割：外部システムの役割の内容説明
```
