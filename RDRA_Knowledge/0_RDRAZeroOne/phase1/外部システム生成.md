# phase1：外部システム
- 「初期要望.txt」から外部システムを洗い出し「外部システム.tsv」を出力する

# 背景理解
- 「RDRA_Knowledge/1_RDRA/RDRA.md」を読込み理解してください
- 初期要望は実現したいビジネスとそれを実現するシステムについて記述している
- ここではRDRAモデルの中の「外部システム」を洗い出す
- 「外部システム」は業務を実現するために連携する外部のシステムを表す
  
## 外部システムの洗出し
- 初期要望からシステムとして必要な外部システムを洗い出す。
- 「業務」「システム」に関わる「外部システム」（システムに関わる外部のシステムの役割）を見つけ、「外部システム」出力フォーマットで出力する
- 外部システムの役割に応じて幾つかの分類に分ける

## 実行ルール
- 初期要望から業務とBUCが導出できないときはヘッダーだけをファイルに出力する
- 出力内容について判断に迷った場合は可能性の一番高いものを選択する

## 出力ルール
- 出力ファイル：「0_RDRAZeroOne/phase1/外部システム.tsv」
- 出力には必ずヘッダー（カラム名）行を出力する
- 空フィールドにするときは、タブ区切りを省略しない
- プログラムは絶対に生成しない。

### 「外部システム」出力フォーマット
出力形式：TSV（タブ区切り）
カラム構成：
1列目: 外部システム群：外部システムの役割に応じて分類する
2列目: 外部システム：システムに関わる外部のシステム。外部システム名は役割が分かる名前にする
3列目: 役割：外部システムの役割の内容説明
ヘッダー行を含め、各フィールドはタブ文字で区切ってください。

## 出力チェックリスト
- [ ] 各行が3カラム（2個のタブ区切り）であることを確認する

## 今すぐ実行
上記の指示に従い、今すぐ「初期要望.txt」を読み込んでファイルを出力してください。
質問や確認は不要です。
