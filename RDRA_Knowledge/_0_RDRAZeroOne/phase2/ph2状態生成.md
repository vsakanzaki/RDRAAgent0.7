# 背景理解
- 「RDRA_Knowledge/_1_RDRA/RDRA.md」を読込み理解してください
- 初期要望は実現したいビジネスとそれを実現するシステムについて記述している
- ここではRDRAモデルの中の状態モデルを洗い出す
- 状態モデルは複数の状態を持つ
  
## 状態モデルの洗出し
- 初期要望からビジネスとして管理すべき状態モデルを洗い出す
- ph1ビジネスポリシーからポリシーを実現するために必要な状態モデルを洗い出す
- ph1要求から要求を実現するために必要な状態モデルを洗い出す
- ph1業務から業務/BUCを横断する状態変化を状態モデルとして洗い出す
- 状態モデル毎に管理すべき状態を洗い出す
- 状態モデルは必ず、複数の状態を持つ
- ビジネスは状態を使って今の状況を把握し、次の状態に遷移させる
-「状態モデル」出力フォーマットで出力する

## 出力ルール
- 出力内容について判断に迷った場合は可能性の一番高いものを選択する
- 出力には必ずヘッダー（カラム名）行を出力する
- 空フィールドにするときは、タブ区切りを省略しない
- プログラムは絶対に生成しない

### 「状態モデル」出力フォーマット
出力先：0_RDRAZeroOne/phase2/ph2状態.tsv
出力形式：TSV（タブ区切り）
カラム構成：
1列目: 状態モデル：状態遷移の管理単位となる
2列目: 状態：状態モデルがとる複数の状態を洗い出す。複数の状態は「,」区切りで一つの文字列として洗い出す
3列目: 説明：状態モデルが必要な理由と管理する状態を説明
- ヘッダー行を含め、各フィールドはタブ文字で区切ってください

## 出力チェックリスト
- [ ] 各行が3カラム（2個のタブ区切り）であることを確認する
