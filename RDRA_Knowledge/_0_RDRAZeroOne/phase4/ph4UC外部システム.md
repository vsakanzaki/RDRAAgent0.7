# phase4：UC-外部システム
- 「0_RDRAZeroOne/phase3/ph3BUC.tsv」を読み込み業務/BUC/アクティビティ/UCの構造とUCにつながる情報、**外部システム**、状態モデル、状態の関係を理解し、UCの役割を深く理解する
- 「BUC.tsv」の記述は上記のファイルのパスを指す
- 「初期要望.txt」を読み外部のシステムと連携するインターフェースになる記述を把握する

# 背景理解
- 「RDRA_Knowledge/_1_RDRA/RDRA.md」を読込みRDRAの構造を理解してください
- 外部システムはイベントを経由してユースケース(UC)と連携する
- 外部システムは役割に応じてUCに関わる

## UCに関わる外部システムからイベントを生成する
- 「ph3BUC.tsv」から各外部システムの役割を把握する
- 「ph3BUC.tsv」からUCが行うことを理解し、関わる外部システムを把握する
- 初期要望.txtにイベントが記述されている場合は、それを名前にする。決して新たな名前を生成しない
- 初期要望.txtにイベントが記述されていない場合は外部システムとUCを連携させるイベントを生成する

### イベント名の命名規則
- イベント名は「UC」と「外部システム」との連携として適切な名前にする
- 「イベント」は「外部システム」と「UC」が連携するインターフェースになる
- そのインターフェースの意図を表す名前にする
- UCと外部システムをつなげた単純な名前にはしない

## 出力ルール
- 出力には必ずヘッダー（カラム名）行を出力する
- 空フィールドにするときは、タブ区切りを省略しない
- プログラムは絶対に生成しない。

### 「UC外部システム」のフォーマット
出力先：0_RDRAZeroOne/phase4/ph4UC外部システム.tsv
出力形式：TSV（タブ区切り）
カラム構成：
1列目: UC：「ph3BUC.tsv」のUCをセットする
2列目: イベント：UCと外部システムが連携するイベント名をセットする
3列目: 「ph3BUC.tsv」の外部システムをセットする
- ヘッダー行を含め、各フィールドはタブ文字で区切ってください。

## 出力チェックリスト
- [ ] 各行が3カラム（2個のタブ区切り）であることを確認する
