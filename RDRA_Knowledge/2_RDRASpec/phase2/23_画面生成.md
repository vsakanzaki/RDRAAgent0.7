# UI定義
RDRA定義を使ってUIを作成する

## 事前知識
- 「RDRA_Knowledge」フォルダーの以下のファイルを参照する
  - 「1_RDRA/RDRA.md」を読込み、RDRA構造を理解する
  - 「1_RDRA/RDRAGraph.md」を読込み、関連データの構造を理解する
  - 「1_RDRA/関連データ.txt」を読込み、定義したRDRAの内容を理解する

- 「2_RDRASpec/論理データモデル.md」を読込み、論理データの構造を理解する
- 「2_RDRASpec/ビジネスルール.md」を読込み、ビジネスルールを理解する

### 関連データから以下の定義を理解する
- コンテキスト：コンテキストとつながるものがシステムのドメインとして管理するものを理解する
- 情報：情報とつながるものから、論理データの作成元になる情報とコンテキストを理解する
- 状態：状態モデル、状態からつながるものからドメインが管理する状態モデルとコンテキストを理解する
- 条件：条件に関わるものからドメインが管理する条件とコンテキストを理解する
- バリエーション：バリエーションにつながるものからドメインで定義するバリエーションとコンテキストを理解する
- 業務、BUC、アクティビティの階層関係を理解する
- アクティビティとUCの関係：UCとBUCの関係をアクティビティ経由で把握する
- UCで出来ること、行うべきことはUCにつながるもので決まる

## 画面の管理単位（画面照会.json）
- 出力するJSONは `画面照会.json` とし、構造は以下に統一する
  - businesses → BUCs → actors → screens
  - actorGroups → actors → screens

### businesses:
- 業務/BUC/アクター別の画面をまとめる

#### BUCs:
- 業務配下にBUCを配置する

##### actors:
- BUC配下にアクター別の画面を配置する

### actorGroups:
- アクター群/アクター別の画面をまとめる

#### actors:
- アクター群の配下にアクター別の画面を配置する

### screens:
- アクターが使う画面の詳細を定義する（項目/操作/データアクセス）

## 画面の仕様を作成する
### 画面の項目を洗い出す（fields）
- 画面の項目はアクターが使える画面を把握する
- 画面に必要な項目は画面につながるUCに必要な項目から決まる
- 画面の項目は画面につながる情報「画面-UC-情報」の項目に依存する

### 画面毎に以下の処理を出力する（画面照会.json）
- 画面に必要な項目を洗い出す
- 画面に関連する全てのアクターに対して、アクター毎に画面（screens）を定義する
- 機能名は `operation`以下の中から適切なものを選んでセットする
  - [一覧,照会,登録,分析,申請,承認]
  - 「機能名の詳細」を参照
- 論理データ(data_access)との関係を設定する
- 画面データ(screen_datas)を生成する

#### 機能名の詳細（operation）
  - 一覧：一覧形式でデータを表示する
  - 照会：現在の状況の把握、データ分析などのデータを表示する
  - 登録：画面から情報を入力し新たな情報を追加、更新、削除する
  - 分析：データを分析する機能をもつ
  - 申請：ワークフローで申請する情報を入力する
  - 承認：ワークフローで登録されたデータを承認/却下する
  
#### データアクセス（data_access）
- 画面の機能に必要なentityを「論理データモデル.md」から洗い出す
- 画面に必要な項目を満たす項目を持つ論理データ(entity)から関係を洗い出す
- 関係するentityとの関係をC,R,U,Dで示す、複数設定可能
  - "C":登録、追加などを表す
  - "R":参照、表示などの表す
  - "U":更新、変更などの表す
  - "D":削除を表す
-　isMain:true(主として操作するEntitiy)/false(主Entitiyから参照されるEntity)
  - 一つの画面に主となるEntityは一つ

#### 画面データ（screen_datas）
- 画面照会.json は画面照会プログラムで使うダミーデータを表す
- screen_datasは画面毎のArrayになっている
- display_nameは１画面分のデータを表し、対応する画面の内容を示す。
  - 例：screen_datas: [{display_name:"田中不動産",dummy_data:[{"項目名": "名前","値": "田中不動産"},{"項目名": "ID","値": "001"}]}]
　- dummy_dataはfieldsの項目名に対応する項目を持つ
- 3画面分のデータを作成する

## 画面照会.json の出力フォーマット
{
  businesses:[{
    business_name:"業務名",
    BUCs:[{
        buc_name:"BUC名",
        actors:[{
            actor_name:"アクター名",
            screen_names:["画面名",]
        }]  // ← 同一BUC内で複数のアクターがいる場合、全てのアクターを出力
    }]
},],
  actorGroups:[{
    group_name:"アクター群",
    actors:[{
      actor_name: "アクター名",
      screen_names: ["画面名",]
    },],
  },]
  screens:[{
    screen_name: "画面名",
    fields: [{"項目名": "項目名",
        type: "string|number|バリエーション",
        tooltip:"項目の説明"
    }],
    screen_datas: [{
        display_name:"表示用のデータ",
        dummy_data:[{"項目名": "項目名","値": "ダミーデータ},]
    }],
    operation:"機能名",
    data_access:[{"entity":entity名,"CRUD":["C","R","U","D"],isMain:true},]  //isMain:true(主Entity)/false(主Entityから参照されるEntity)
},]
}
