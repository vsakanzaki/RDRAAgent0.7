# UI定義
RDRA定義を使って画面とBUC、アクターの関係を作成する
画面一覧.jsonをベースに画面が操作するデータを追記し、画面と業務・BUCの関係、アクターと画面の関係を追記し画面照会.jsonを作成する

## 実施すること
- 「BUC画面.json」を読込み「businesses」をそのまま出力する
- 「アクター画面.json」を読込み「actorGroups」をそのまま出力する
- 「画面一覧.json」を読込み「screens」にデータアクセス「data_access」を追記して出力する
- 上記３つを一つにまとめ「画面照会.json」として出力する

## screens:　画面にデータアクセスを追記する
- 「画面一覧.json」の「screens:」に「data_access」を追加し「画面照会.json」を出力する
  - 論理データ(data_access)との関係を追記する
- 画面が操作する「データアクセス」を追加する
- データは「論理データモデル.md」のEntityから選択する
- 画面とEntitiyの関係は関係データから以下の関係データを頼りに類推する
  - 画面 → UC → 情報
  - 情報と論理データの関係を類推する

### データアクセス（data_access）
- 画面の機能に必要なentityを「論理データモデル.md」から洗い出す
- 画面に必要な項目を満たす項目を持つ論理データ(entity)から関係を洗い出す
- 関係するentityとの関係をC,R,U,Dで示す、複数設定可能
  - "C":登録、追加などを表す
  - "R":参照、表示などの表す
  - "U":更新、変更などの表す
  - "D":削除を表す
-　isMain:true(主として操作するEntitiy)/false(主Entitiyから参照されるEntity)
  - 一つの画面に主となるEntityは一つ

## 画面照会.json の出力フォーマット
{
  "businesses":[{
    "business_name":"業務名",
    BUCs:[{
        "buc_name":"BUC名",
        "actors":[{
            "actor_name":"アクター名",
            "screen_names":["画面名",]
        }]  // ← 同一BUC内で複数のアクターがいる場合、全てのアクターを出力
    }]
  },],
  "actorGroups":[{
    "group_name":"アクター群",
    "actors":[{
      "actor_name": "アクター名",
      "screen_names": ["画面名",]
    },],
  },]
  "screens":[{
    "screen_name": "画面名",
    "fields": [{"field_name": "項目名",
        "type": "string|number|バリエーション",
        "tooltip":"項目の説明"
    }],
    "screen_datas": [{
        "display_name":"表示用のデータ",
        "dummy_data":[{"field_name": "項目名","値": "ダミーデータ"},]
    }],
    "operation":"機能名",
    "data_access":[{"entity":entity名,"CRUD":["C","R","U","D"],"isMain":true},]  //isMain:true(主Entity)/false(主Entityから参照されるEntity)
},]
}
