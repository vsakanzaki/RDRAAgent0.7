# UI定義
関連データを使って画面と業務/BUC、アクターとの関係を作成する
画面一覧.jsonをベースに画面が操作するデータを追記し、画面と業務・BUCの関係、アクターと画面の関係を追記しBUC画面.jsonを作成する

## 事前知識
- 「RDRA_Knowledge/1_RDRA/RDRA.md」を読込み、RDRA構造を理解する
- 「RDRA_Knowledge/1_RDRA/RDRAGraph.md」を読込み、関連データの構造を理解する
- 「1_RDRA/関連データ.txt」を読込み、定義したRDRAの内容を理解する

### 関連データから以下の定義を理解する
- 業務の階層構造：業務、BUC、アクティビティは階層構造を持つ
- 画面はアクティビティにつながるUCとのつながりから画面が使われる環境を理解する

### businesses:
- 「関連データ.txt」から全「業務」を抽出する

#### BUCs:
- 「関連データ.txt」から「業務」に関連する「BUC」を抽出する

##### actors:
- 以下の関連を元に「関連データ.txt」から「アクター」を抽出する
  - 業務 → BUC → アクティビティ → UC → 画面 → アクター

##### screen_names:
- 「関連データ.txt」から「アクター」につながる「画面」を抽出する
- 画面は「画面一覧.json」で定義されているものを使う
- 決して架空の画面を生成しない

## BUC画面.json の出力フォーマット
{
  "businesses":[{
    "business_name":"業務名",
    BUCs:[{
        "buc_name":"BUC名",
        "actors":[{
            "actor_name":"アクター名",
            "screen_names":["画面名",]
        }]  // ← 同一BUC内で複数のアクターがいる場合、全てのアクターを出力
    }]
  },]
}
