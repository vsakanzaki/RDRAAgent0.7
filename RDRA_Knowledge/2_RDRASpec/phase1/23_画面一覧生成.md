### 関連データから画面に関わる以下の定義を理解する  ※画面：RDRA定義上の画面を表す
- 業務、BUC、アクティビティの階層関係を理解する
- アクティビティとUCの関係：UCとBUCの関係をアクティビティ経由で把握する
- 画面がどのような状況で使われるかは、画面につながるUCとアクティビティから類推できる
- 画面はUCとアクターのインターフェースとなる
- 「ビジネスルール.md」の中の条件は、RDRAの条件の詳細を表す
- 画面の項目にバリエーションがある時は以下の関連からビジネスルールの詳細な条件が画面に影響を受ける
  - 画面 → UC → 条件 → バリエーション
- 以下の関係から画面の機能を理解する
  - 画面はUCを経由して情報を操作しシステムの整合性を確保する
  - 画面とバリエーションの関係は以下の関係から導ける
    - 画面 → UC → 条件 → バリエーション
    - 以下の場合に画面でバリエーションの値を選択する
      - 画面 → UC → 条件 → バリエーション and 画面 → UC → 情報にバリエーションがない
  - 画面はUCを経由して状態を変える
    - 状態を変えるために必要な入力を画面で行う
  - イベントはUCと外部システムとの連携する

### screens:
- 画面の詳細を定義する
  - 項目(fields)
  - 画面データ(screen_datas)
  - 機能

## 画面の仕様を作成する
### 画面毎に以下の処理を出力する（画面一覧.json）
- 画面に必要な項目を洗い出す
- 機能名は `operation`以下の中から適切なものを選んでセットする
  - [一覧,照会,登録,分析,申請,承認]
  - 「機能名の詳細」を参照
- 画面データ(screen_datas)を３件生成する

### 画面の項目を洗い出す（fields）
- 画面に必要な項目は画面につながるUCに必要な項目から決まる
- 画面の項目は以下の観点で洗い出す
  - 画面と関係する情報の項目：画面 → UC → 情報
  - 画面につながるアクターの役割から必要な項目
  - 画面につながるUCの役割から必要な入力項目
- 画面の項目
  - typeの決め方
    - バリエーションや分類、状態モデル、状態などの複数の値を持つ場合は「enum」を設定する
    - 名前や住所などの場合は「string」を設定する
    - 金額や数量などの場合は「number」を設定する
    - 上記以外は「string」を設定する
  - tooltip:
    - バリエーションや分類、状態モデル、状態などの複数の値を持つ場合は値を設定する
    - 上記以外は項目の説明を設定する
- 画面の項目は画面につながる情報「画面-UC-情報」の項目に依存する

#### 機能名の詳細（operation）
  - 一覧：一覧形式でデータを表示する
  - 照会：現在の状況の把握。データを表示する
  - 登録：画面から情報を入力し新たな情報を追加、更新、削除する
  - 分析：データを分析する機能をもつ
  - 申請：ワークフローで申請する情報を入力する
  - 承認：ワークフローで登録されたデータを承認/却下する

#### 画面データ（screen_datas）
- 「画面一覧.json」 は画面照会プログラムで使うダミーデータを表す
- screens:は複数画面のArrayである
- screen_nameは画面の名前である
- fields:画面項目
  - type:画面のTypeは画面に関わる情報の属性のTypeから導出する
- screen_datasは画面毎に複数データを持つ
  - display_nameは１画面分のデータを表し、対応する画面の内容を示す。
    - 例：screen_datas: [{"display_name":"田中不動産","dummy_data":[{"field_name": "名前","value": "田中不動産"},{"field_name": "ID","value": "001"}]}]
　- dummy_dataはfieldsの項目名に対応する項目を持つ
  - 3件のデータを作成する

## 「画面一覧.json」の出力フォーマット
{
  "screens":[{
    "screen_name": "画面名",
    "fields": [{
        "field_name": "項目名",
        "type": "string|number|enum",
        "tooltip":"項目の説明"
    }],
    "screen_datas": [{
        "display_name":"表示用のデータ",
        "dummy_data":[{"field_name": "項目名","value": "値"},]
    }],
    "operation":"機能名",
  },]
}
