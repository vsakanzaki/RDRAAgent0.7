# web_tool（ブラウザUI系ツール）

このフォルダーは、RDRAAgent が生成する仕様データ（主に `2_RDRASpec/画面照会.json`）を**ブラウザで「閲覧」するための簡易Webツール**を置く場所です。  
Node.js の標準モジュールのみで動作し、追加の依存関係はありません。

---

## できること（概要）

- **BUC → アクター → 画面**、または **アクター群 → アクター → 画面** の導線で、画面定義をツリーで辿って閲覧できます
- 画面ごとに以下を確認できます
  - **機能種別**（例: 登録/照会/分析/未定義）
  - **fields**（項目一覧。ダミーデータから初期値表示）
  - **screen_datas**（画面データ行の切替）
  - **data_access**（エンティティ別 CRUD、MAIN 表示）

---

## ファイル構成

- `bucActorUI.html`
  - ブラウザで動く閲覧UI本体（単体HTML）
  - 同一オリジンの `GET /ui.json` からデータを読み込みます
  - 右上の「閉じる」から `GET /shutdown` を呼び、サーバ停止を試みます

- `bucActorUI.js`
  - `bucActorUI.html` を配信する簡易HTTPサーバ
  - 追加で `GET /ui.json` を提供します（入力は `2_RDRASpec/画面照会.json` を優先）
  - `GET /shutdown` でサーバを停止できます（UIの「閉じる」ボタンから呼ばれます）
  - ポートは **3002** 固定（`PORT = 3002`）

- `bucActorUI.test.html`
  - `bucActorUI.html` の主要イベント（タブ切替、ツリー選択、カード選択、データ行選択、閉じる等）を
    **ブラウザ単体で網羅的に動作確認**するための簡易テストランナー
  - `fetch` や `confirm/alert/window.close` をスタブ化して、iframe（`srcdoc`）内でUIを動かして検証します

---

## 使い方（閲覧UIの起動）

### 前提

- 仕様データ `2_RDRASpec/画面照会.json` が存在すること
  - 無い場合は、まず RDRAAgent のメニュー（例: 仕様生成）で作成してください
  - サーバ側は `2_RDRASpec/画面照会.json` を優先し、無ければ `2_RDRASpec/ui.json` を探します

### 起動

リポジトリルートで以下を実行します。

```bash
node RDRA_Knowledge/helper_tools/web_tool/bucActorUI.js
```

起動するとコンソールにURLが表示されます。ブラウザで以下へアクセスします。

- `http://localhost:3002/`

### 停止

- ブラウザ右上の「閉じる」ボタン（`/shutdown` を呼びます）
- もしくはターミナルで `Ctrl + C`

## 画面（UI）の見方

- **上部タブ**
  - **BUC**: 業務 → BUC → アクター のツリーで辿って画面を閲覧
  - **アクター**: アクター群 → アクター のツリーで辿って画面を閲覧

- **左ツリー**
  - アクターを選択すると、中央（画面一覧）と右（画面詳細）が表示されます
  - 業務/BUC/アクター群のノードを選んだ状態では、画面一覧はブランク表示になります（「アクターを選択してください」）

- **中央（画面一覧）**
  - 選択アクターに紐づく `screen_names` がカードで表示されます
  - 画面が `screens` に未定義でも、プレースホルダーとして「未定義」で表示されます

- **右（画面詳細）**
  - `screen_datas` の行をクリックして、フォームの初期値（ダミーデータ）を切り替えられます
  - `data_access` があれば、エンティティとCRUD、MAINを表示します

## テスト（イベント網羅テスト）

`bucActorUI.test.html` は、**サーバを立てずに**ブラウザで開いて実行できます。

1. `RDRA_Knowledge/helper_tools/web_tool/bucActorUI.test.html` をブラウザで開く
2. 画面上部が `PASS` になれば成功

補足:

## トラブルシューティング

- **`2_RDRASpec/画面照会.json（または ui.json）が存在しません`**
  - 仕様データが未生成です。RDRAAgent 側の仕様生成を先に実施してください。

- **`ポート 3002 は既に使用されています（EADDRINUSE）`**
  - 既に別プロセスが 3002 を使用しています。該当プロセスを終了してから起動してください。

- **UIが「データの読み込みに失敗しました」になる**
  - `GET /ui.json` が失敗している可能性があります。サーバが起動しているか、`2_RDRASpec/画面照会.json` が読めるか確認してください。
