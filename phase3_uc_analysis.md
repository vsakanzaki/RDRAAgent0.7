# Phase3 UC洗出し分析

## 第1パス：業務起点のUC洗出し

### 1. 会員を登録するフロー ✓
- UC1: 「会員基本情報を登録する」
  - 対象アクティビティ: 会員情報担当者が基本情報を登録する
  - 情報: 介護会員情報
  - アクター: 会員情報担当者
  - 状態: 会員ステータス (→登録済)
  - 説明: 入力された基本情報を会員データベースに登録し、会員IDを付与する

### 2. 会員情報を更新するフロー ✓
- UC2: 「会員属性情報を更新する」
  - 対象アクティビティ: 会員情報担当者が属性情報を更新する
  - 情報: 介護会員情報
  - アクター: 会員情報担当者
  - 状態: 会員ステータス (登録済/利用中/休止→利用中/休止/終了)
  - 説明: 会員の属性情報を更新し、会員情報を最新に保つ

### 3. スケジュールを計画するフロー ✓
- UC3: 「訪問スケジュールを計画する」
  - 対象アクティビティ: スケジュール管理者がスケジュールを計画する
  - 情報: スケジュール情報,介護会員情報,サービススタッフ情報
  - アクター: スケジュール管理者
  - 状態: スケジュール計画状態 (計画待機→計画確定)
  - 説明: 介護会員のニーズとスタッフのスキル・働き方に基づいて訪問スケジュールを計画する

### 4. スケジュールを調整するフロー ✓
- UC4: 「訪問スケジュールを調整する」
  - 対象アクティビティ: スケジュール管理者がスケジュールを再調整する
  - 情報: スケジュール情報
  - アクター: スケジュール管理者
  - 状態: スケジュール計画状態 (計画確定,調整中→最終確定)
  - 説明: 会員やスタッフの要望変化に対応してスケジュールを調整する

### 5. 訪問介護を実施するフロー
- UC5: 「訪問実施を開始する」
  - 対象アクティビティ: サービススタッフが訪問介護を実施する
  - 情報: スケジュール情報,サービススタッフ情報
  - アクター: サービススタッフ
  - 状態: 訪問実施状態 (予定→実施中)
  - 説明: スケジュール予定に基づいて計画された訪問介護を実施開始

### 6. 実施記録を作成するフロー ✓
- UC6: 「訪問実施記録を作成する」
  - 対象アクティビティ: サービススタッフが訪問介護の内容を記録する
  - 情報: 訪問介護実施記録
  - アクター: サービススタッフ
  - 状態: 実施記録状態 (未作成→作成中)
  - 説明: 実施した訪問介護の内容を記録する

- UC7: 「訪問実施記録を登録する」
  - 対象アクティビティ: スケジュール管理者が実施記録をデータベースに登録する
  - 情報: 訪問介護実施記録
  - アクター: スケジュール管理者
  - 状態: 実施記録状態 (確認待機→確定)
  - 説明: 確認した実施記録をデータベースに登録する

### 7. 請求を作成するフロー ✓
- UC8: 「費用を計算する」
  - 対象アクティビティ: 請求担当者が費用を計算する
  - 情報: 訪問介護実施記録,介護会員情報
  - アクター: 請求担当者
  - 状態: 請求状態 (未計算→計算完了)
  - 説明: 実施記録に基づいて介護会員に対する費用を計算する

- UC9: 「請求書を作成する」
  - 対象アクティビティ: 請求担当者が請求書を作成する
  - 情報: 請求情報
  - アクター: 請求担当者
  - 状態: 請求状態 (計算完了→請求待機)
  - 説明: 計算した費用に基づいて請求書を作成する

### 8. 回収を行うフロー
- UC10: 「支払いを確認する」
  - 対象アクティビティ: 請求担当者が支払い確認を行う
  - 情報: 請求情報
  - アクター: 請求担当者
  - 状態: 請求状態 (請求済,支払待機→回収済)
  - 説明: 介護会員からの支払いを確認する

- UC11: 「支払い記録を登録する」
  - 対象アクティビティ: 請求担当者が支払い記録を登録する
  - 情報: 請求情報
  - アクター: 請求担当者
  - 外部システム: 決済システム
  - 説明: 確認した支払いを会計管理システムに記録する

## 第2パス：状態遷移による網羅性チェック

### 会員ステータス: 登録待機, 登録済, 利用中, 休止, 終了
- →登録済: UC1「会員基本情報を登録する」✓
- 登録済→利用中: UC2「会員属性情報を更新する」(From: 登録済, To: 利用中)
- 利用中→休止: UC2「会員属性情報を更新する」(From: 利用中, To: 休止)
- 休止→利用中: UC2「会員属性情報を更新する」(From: 休止, To: 利用中)
- 利用中→終了: UC2「会員属性情報を更新する」(From: 利用中, To: 終了)

**判定**: UC2が複数の状態遷移に対応。状態遷移は網羅 ✓

### スケジュール計画状態: 計画待機, 計画中, 計画確定, 調整中, 最終確定
- 計画待機→計画確定: UC3「訪問スケジュールを計画する」✓
- 計画確定→調整中: UC4「訪問スケジュールを調整する」(1段階目)
- 調整中→最終確定: UC4「訪問スケジュールを調整する」(2段階目)

**判定**: 「計画中」状態への遷移がUCで明示されていない。ただし、計画待機→計画確定で進むため、計画中は中間状態の可能性がある。現在のUCで遷移を網羅できると判定 ✓

### スタッフ勤務状態: 登録待機, 登録済, 稼働中, 休職, 退職
- →登録済: **不足** → 新BUC「スタッフ情報管理業務」/UC「スタッフ情報を登録する」を追加
- 登録済→稼働中: **不足** → UC「スタッフの稼働を開始する」を追加
- 稼働中→休職: **不足** → UC「スタッフを休職する」を追加
- 休職→稼働中: **不足** → UC「スタッフの復帰を受け付ける」を追加
- 稼働中→退職: **不足** → UC「スタッフを退職する」を追加

**判定**: 新規業務追加が必要

### 訪問実施状態: 予定, 実施中, 実施完了, キャンセル, 未実施
- →予定: スケジュール計画で設定（UC3の結果）
- 予定→実施中: UC5「訪問実施を開始する」✓
- 実施中→実施完了: **不足** → UC「訪問実施を完了する」を追加
- 実施中→キャンセル: **不足** → UC「訪問実施をキャンセルする」を追加
- 予定→未実施: **不足** → UC「訪問実施を未実施にする」を追加
- 予定→キャンセル: **不足** → UC「訪問実施をキャンセルする」を追加

**判定**: 複数のUCを追加が必要

### 実施記録状態: 未作成, 作成中, 確認待機, 確定, 請求済
- →未作成: 訪問実施完了で自動設定
- 未作成→作成中: UC6「訪問実施記録を作成する」✓
- 作成中→確認待機: UC6の完了で遷移
- 確認待機→確定: UC7「訪問実施記録を登録する」✓
- 確定→請求済: UC8「費用を計算する」の結果で遷移

**判定**: 状態遷移を網羅 ✓

### 請求状態: 未計算, 計算中, 計算完了, 請求待機, 請求済, 支払待機, 回収済
- →未計算: 月末に自動設定
- 未計算→計算完了: UC8「費用を計算する」✓
- 計算完了→請求待機: UC9「請求書を作成する」✓
- 請求待機→請求済: **不足** → UC「請求書を送付する」を追加？
- 請求済→支払待機: 自動遷移
- 支払待機→回収済: UC10「支払いを確認する」✓

**判定**: 「請求書を送付する」UCの追加が必要

### 事業所管理状態: 管理中, 新設準備, 統合予定, 統合実行, 終了予定, 終了
- **不足**: 関連するBUC/UCがない → 新BUC「事業所管理業務」を追加

**判定**: 新規業務追加が必要

## 追加が必要なUC一覧

### スタッフ情報管理業務（新規BUC）
- UC: スタッフ基本情報を登録する（登録待機→登録済）
- UC: スタッフの稼働を開始する（登録済→稼働中）
- UC: スタッフを休職する（稼働中→休職）
- UC: スタッフの復帰を受け付ける（休職→稼働中）
- UC: スタッフを退職する（稼働中→退職）

### 訪問スケジュール管理業務（既存BUC拡張）
- UC: 訪問実施を完了する（実施中→実施完了）
- UC: 訪問実施をキャンセルする（実施中/予定→キャンセル）
- UC: 訪問実施を未実施にする（予定→未実施）

### 請求・回収業務（既存BUC拡張）
- UC: 請求書を送付する（請求待機→請求済）

### 事業所管理業務（新規BUC）
- UC: 事業所の新設準備を開始する（管理中→新設準備）
- UC: 事業所の統合を計画する（新設準備→統合予定）
- UC: 事業所の統合を実行する（統合予定→統合実行→管理中）
- UC: 事業所の終了を計画する（管理中→終了予定）
- UC: 事業所を終了する（終了予定→終了）
